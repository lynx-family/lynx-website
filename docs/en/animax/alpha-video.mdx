# Alpha Video

AnimaX supports rendering Alpha Video layers, which is an extension to the Lottie specification. It allows a layer in the animation to be played as transparent video, effectively reducing performance overhead while remaining compatible with all animation features.

As an animation framework, AnimaX may not support the latest advanced After Effects features. For incompatible features, convert the layer to an Alpha Video layer so it can be played with the intended effect.

When using assets that contain Alpha Video layers, no extra code configuration is required. Playback is the same as playing a regular AnimaX asset.

## Key advantages

To cover all effects, a common solution is to export every rendered frame of a layer as an image sequence and render it with image layers. Compared with that approach, Alpha Video produces smaller assets and has better memory performance.

Alpha Video layers are embedded into the animation as independent layers and can form hierarchy, blending, mask, and parent-child relationships with other layers. At the same time, you still get AnimaX playback control and interactivity, enabling more flexible composition and control.

### Run compatibility checks for all layers

Click Compatibility Check under each pre-composition to view the compatibility report.

For the pre-composition corresponding to Alpha Video, check whether the layer dimensions are multiples of 16; if not, adjust to an integer multiple of 16 to prevent black borders in the video.

Meanwhile, the compatibility check for Alpha Video provides information about whether each layer needs to be converted to Alpha Video. If a layer has no related error reports during the compatibility check, there is no need to convert it to Alpha Video. Based on this, and according to performance needs, you can optimize [Alpha Video size](#optional-optimize-alpha-video-size).

For the pre-composition corresponding to non-Alpha Video layers, focus on error information other than layer size. Use the locate button to jump to the relevant position, disable or remove unsupported features, and then preview the animation again. If the animation meets expectations, keep it as is. If it does not, consider implementing it using other AnimaX-supported methods or converting it to Alpha Video.

## How it works

Alpha Video relies on transparent video assets for playback. A transparent video is a type of video that has an alpha channel. In the figure below, the left area is the transparent channel of this video.

<img
  src="https://lf-lynx.tiktok-cdns.com/obj/lynx-artifacts-oss-sg/plugin/static/animax_alpha_video_channel.png"
  width="1397px"
  height="688px"
/>

In the json file, metadata for transparent video assets and Alpha Video is managed. They are declared and indexed similarly to image assets.

```json {12,30,32}
{
  ...
  assets:[
    ...
  ],
  fonts:{
    ...
  },
  videos:[
    ...
    {
      "id": "video_1",
      "x": 704,
      "y": 0,
      "w": 704,
      "h": 704,
      "ax": 0,
      "ay": 0,
      "aw": 704,
      "ah": 704,
      "u": "videos/",
      "p": "video_1.mp4",
      "sz": 863796
    }
  ],
  layers:[
    ...
    {
      ...
      "ty": 1009,
      "nm": "content",
      "refId": "video_1",
    }
    ...
  ]
}
```

:::details What are "ax" and "ay"?

In a transparent video, the alpha channel exists as an independent content area. The parameters "ax", "ay", "aw", and "ah" describe the position and size of that alpha channel within the transparent video.

In general, "aw" will be half the original video width. If you plan to create transparent videos yourself or modify the json file, ensure that the metadata matches the video resolution and correctly describes the channel position.

:::

During playback, Alpha Video parses the asset, extracts the per-frame video texture, composites the transparent layer with the video, and converts the result into an image. This image is then fed into the rendering engine and composited with other layers to produce the final frame.

<img
  src="https://lf-lynx.tiktok-cdns.com/obj/lynx-artifacts-oss-sg/plugin/static/animax-alpha-video-process.png"
  width="2506px"
  height="766px"
/>

:::warning Use Alpha Video with care

- ⚠️ Every frame of Alpha Video requires texture updates and image composition. Overusing Alpha Video can reduce playback smoothness.

- ⚠️ Video width and height need to be integer multiples of 16 to meet decoding alignment requirements. Alpha Videos that do not meet this requirement will produce black borders.

:::
