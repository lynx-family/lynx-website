# Tutorial: Playback Timeline

Learn timeline-level playback controls: limiting the playable frame range, changing speed, setting initial progress, playing a segment, and querying playback state from the runtime.

Use `start-frame` and `end-frame` when only part of the source animation should be played. Use `speed` for the playback rate, `progress` for the initial position, and `loop-count` or `bindrepeat` / `bindcompletion` when the UI needs to reflect repeat and completion state.

```tsx {5-11}
<animax-view

```

`getDuration`, `getCurrentFrame`, and `isAnimating` return runtime state through the UI Method callback. As with playback methods, call them after `ready`.

```tsx {1-15,20-23}
function invokeQuery(method: QueryMethod, onResult: (value: string) => void) {
  lynx
    .createSelectorQuery()
    .select(`#${ANIMAX_VIEW_ID}`)
    .invoke({
      method,
      success(res) {
        onResult(formatQueryResult(method, res));
      },
    })
    .exec();
}

const queryCurrentFrame = () => {
  invokeQuery('getCurrentFrame', setFrame);
};

<animax-view
  id={ANIMAX_VIEW_ID}
  src={METHODS_QUERY_LOTTIE_SRC}
  autoplay={true}
  bindready={handleReady}
/>;
```
