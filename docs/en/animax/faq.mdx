# FAQ

This page covers common rendering, control, asset, and font issues when using AnimaX.

## Rendering issues

### Some frames are skipped or not rendered

Mobile rendering is usually synchronized to a frame interval around 16 ms for 60 FPS. If the main thread is busy, the actual interval can become longer, so some frames may not be rendered.

If an animation relies on many very short frames, increase each frame's duration where possible. If it uses many image-sequence frames, consider converting suitable parts to [Alpha Video](./alpha-video).

### Alpha Video shows black borders

For Alpha Video, make sure the video width and height are integer multiples of 16. Some decoders require this alignment and can pad non-aligned videos, causing black borders.

If borders still appear after using aligned dimensions, expand the source video by a few transparent pixels so the padded area remains transparent.

## Control issues

### The animation stops unexpectedly

If the animation is mounted inside a popup or overlay, try setting [`ignore-attach-status`](/api/elements/built-in/animax-view.html#ignore-attach-status) to `true`. If the issue is lifecycle-related, also check [`ignore-lynx-lifecycle`](/api/elements/built-in/animax-view.html#ignore-lynx-lifecycle).

If playback still stops, check whether business logic calls `pause`, `stop`, or `cancel`, and use [`bindcancel`](/api/elements/built-in/animax-view.html#bindcancel) to help trace cancellation.

### `autoplay={false}` is set and manual `play` does not start

Call `play` after the [`bindready`](/api/elements/built-in/animax-view.html#bindready) callback. The animation can only be played after the resource has loaded and parsed.

### Poster image switches to animation with stretching

Keep the poster image display mode aligned with the AnimaX fit mode. For example, `objectfit="cover"` should use an image mode that also fills the container proportionally.

```tsx
const [isFrameReady, setIsFrameReady] = useState(false);

const handleFirstFrame = () => {
  setIsFrameReady(true);
};

return (
  <view className="container">
    <animax-view
      className="animax-item"
      bindfirstframe={handleFirstFrame}
      src="https://example.com/animation.json"
      objectfit="cover"
    />
    {!isFrameReady && (
      <image
        className="poster-item"
        src="https://example.com/poster.png"
        mode="aspectFill"
      />
    )}
  </view>
);
```

See [`objectfit`](/api/elements/built-in/animax-view.html#objectfit) for details.

## Asset issues

### Animation JSON and image assets are not bundled

When using local files, make sure the assets are included in your build output and that the animation JSON references the final deployed asset paths. See Rspeedy output and asset configuration for details.

### Font file loading fails

Check the `origin` value under `fonts.list` in the Lottie JSON.

```json
{
  "fonts": {
    "list": [
      {
        "origin": 3,
        "fPath": "",
        "fFamily": "Example Font"
      }
    ]
  }
}
```

In the [Lottie specification](https://lottiefiles.github.io/lottie-docs/constants/#font-path-origin), `origin: 3` means the font is loaded from a remote resource. Make sure `fPath` is a valid URL, or set the font resource through Dynamic Properties before playback starts.

If you only need a system fallback font, use `origin: 0` so the runtime does not request a remote font file.

### Custom fonts are needed for text layers

Use the AFX export flow when possible. If you need to adjust the JSON manually, set `origin` and `fPath` explicitly.

```json
{
  "fonts": {
    "list": [
      {
        "origin": 3,
        "fPath": "https://example.com/fonts/example.ttf",
        "fFamily": "Example Font"
      }
    ]
  }
}
```

## Other issues

### iOS implicit animation affects playback

Set [`implicit-animation={false}`](/api/elements/built-in/animax-view.html#implicit-animation) when implicit native animations interfere with AnimaX playback.
