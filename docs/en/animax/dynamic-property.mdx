# Dynamic Properties

In Lynx apps, AnimaX provides runtime updates to animation properties for frontend developers via JS N-API.

:::info

Lynx component examples on this page use [`<animax-view>`](/api/elements/built-in/animax-view.html) integration. Native app integrations can use the Android and iOS APIs directly.

Animation properties are divided into Layer Properties and Resource Properties. Layer properties include transform, visibility, etc., and can be modified dynamically during playback. Resource properties include image, text, and video assets, and can only be modified before playback starts.

:::

## Use dynamic properties via frontend components

We define the dynamic property types and wrap the related APIs.

In practice, it’s recommended to keep dynamic property calls behind a typed frontend component so resource updates, layer updates, and playback lifecycle handling stay in one place.

### Submit resource properties

:::info

Before resource properties are submitted, any attempt to render animations (such as `seek` or `resume`) is undefined behavior.

In addition to explicitly submitting resource properties, you can also submit them by calling `play` or `playSegment`. Explicit submission allows you to preload resources and render a specific frame in advance, enabling seamless instant display in popup scenarios.

:::

### Use wildcards to update layer properties

AnimaX supports wildcard updates to layer properties. When `layerName` is `**`, AnimaX attempts to update the matching property for all layers, while `*` matches all layers at one hierarchy level.
When duplicated layer names are provided in layerName, all matching layers will be selected for property updates.
