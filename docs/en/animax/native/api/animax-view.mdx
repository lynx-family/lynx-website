# AnimaXView

`AnimaXView` is the native playback view. It owns the rendering surface, loads animation resources, and exposes playback controls, event listeners, and dynamic property updates.

## Initialization

### initWithContext

Creates a playback view with an `AnimaXContext`. Use it when you need to inject a custom ability, services, or runtime options through the context.

**Parameters**

| Name      | Type              | Description                 |
| --------- | ----------------- | --------------------------- |
| `context` | `AnimaXContext *` | Animation instance context. |

**Returns**

`AnimaXView *`

### initWithPlayer

Creates a playback view with an existing `AnimaXPlayer`. Use it when the player and view lifecycles need to be managed separately.

**Parameters**

| Name     | Type             | Description                 |
| -------- | ---------------- | --------------------------- |
| `player` | `AnimaXPlayer *` | Underlying player instance. |

**Returns**

`AnimaXView *`

## Properties

### enableNativeTapLayerEvent

Controls whether native touch events trigger layer hit testing. When enabled, matched layer names are delivered through `onTapLayers:`.

**Type**

`BOOL`

### ignoreAttachStatus

Controls whether view attachment state participates in surface initialization and visibility updates. Use it only when the host view lifecycle needs special handling.

**Type**

`BOOL`

### player

Read-only access to the underlying player protocol object.

**Type**

`id<AnimaXPlayerProtocol>`

## Resource Loading

### setSrc

Loads an animation resource. Use the URL-only form for a direct resource URL, or pass a bundle when dependent resources such as images should be resolved from that bundle.

**Available Signatures**

| Signature          | Description                                |
| ------------------ | ------------------------------------------ |
| `setSrc:`          | Loads an animation from a URL string.      |
| `setSrc:inBundle:` | Loads an animation from a specific bundle. |

**Parameters**

| Name     | Type         | Description                     |
| -------- | ------------ | ------------------------------- |
| `src`    | `NSString *` | Animation resource URL.         |
| `bundle` | `NSBundle *` | Bundle containing the resource. |

### setJson

Loads an animation from a JSON string. Use the bundle overload when dependent resources should be resolved from a specific bundle.

**Available Signatures**

| Signature           | Description                                                                                       |
| ------------------- | ------------------------------------------------------------------------------------------------- |
| `setJson:`          | Loads an animation from a JSON string.                                                            |
| `setJson:inBundle:` | Loads an animation from a JSON string and resolves dependent resources from the specified bundle. |

**Parameters**

| Name     | Type         | Description                            |
| -------- | ------------ | -------------------------------------- |
| `json`   | `NSString *` | Lottie JSON string.                    |
| `bundle` | `NSBundle *` | Bundle containing dependent resources. |

### setComposition

Sets an already parsed `AnimaXComposition`. Use it to reuse a preloaded composition.

**Parameters**

| Name          | Type                  | Description            |
| ------------- | --------------------- | ---------------------- |
| `composition` | `AnimaXComposition *` | Animation composition. |

### setImageFolder

Sets the directory for external image assets. Call it before `setJson:` or `setSrc:`.

**Parameters**

| Name          | Type         | Description            |
| ------------- | ------------ | ---------------------- |
| `imageFolder` | `NSString *` | Image asset directory. |

### setDynamicResource

Enables or disables dynamic resource updates during playback.

**Parameters**

| Name     | Type   | Description                               |
| -------- | ------ | ----------------------------------------- |
| `enable` | `BOOL` | `YES` to enable dynamic resource updates. |

## Constructors

### AnimaXView

Creates a playback view. Pass only `Context` when creating it in code, or pass `Context` and `AttributeSet` when inflating from XML and reading `AnimaXView` XML attributes.

**Available Signatures**

| Signature                                         | Description                                               |
| ------------------------------------------------- | --------------------------------------------------------- |
| `AnimaXView(Context context)`                     | Creates a playback view with the default `NativeAbility`. |
| `AnimaXView(Context context, AttributeSet attrs)` | Creates a playback view from XML layout.                  |

**Parameters**

| Name      | Type           | Description        |
| --------- | -------------- | ------------------ |
| `context` | `Context`      | Android context.   |
| `attrs`   | `AttributeSet` | XML attribute set. |

**Returns**

`AnimaXView`

## XML Attributes

### app:animax_animationUrl

Sets the animation resource URL. Maps to `setSrc(String src)`.

### app:animax_autoPlay

Sets whether playback starts automatically after loading. Maps to `setAutoPlay(boolean autoPlay)`. Defaults to `true`.

### app:animax_loop

Sets whether playback loops. Maps to `setLoop(boolean enable)`. Defaults to `false`.

### app:animax_objectFit

Sets content fitting. Maps to `setObjectFit(ObjectFit objectFit)`. Defaults to `contain`.

## View Options

### getPlayer

Returns the `IAnimaXPlayer` held by the view. Use it when the host wants to manage resource loading, playback controls, or event listeners through the player interface.

**Returns**

`IAnimaXPlayer`

### setIgnoreAttachStatus

Controls whether the view ignores Android attach state. Enable it when the host already manages visibility or attachment state and needs AnimaX to create a temporary surface without relying on the normal attach lifecycle.

**Parameters**

| Name     | Type      | Description                    |
| -------- | --------- | ------------------------------ |
| `ignore` | `boolean` | `true` to ignore attach state. |

### setEnableTapLayerEvent

Controls whether native layer tap detection is enabled. When enabled, `onTouchEvent` forwards the touch point to AnimaX and reports matched layer names through `onTapLayers(AnimaXTapParam param)`.

**Parameters**

| Name     | Type      | Description                        |
| -------- | --------- | ---------------------------------- |
| `enable` | `boolean` | `true` to enable layer tap events. |
