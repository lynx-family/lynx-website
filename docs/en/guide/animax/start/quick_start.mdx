# Quick Start

In this document, we will start from a Lynx project to run and verify a minimal runnable example of AnimaX.

### Prepare the Animation

Typically, designers export Lottie resources from AE using the AFX plugin.
We also provide a sample resource for demonstration, which you can use directly.

  Download sample resources
</a>

If you’re interested in how this resource is produced, see [Plugin Export](../start/afx.mdx) to learn the AFX export process and specifications.

::: details How does AnimaX organize Lottie animation assets?

For any animation resource, there is an accompanying `.json` file conforming to the [Lottie specification](https://lottiefiles.github.io/lottie-docs/) that stores metadata for the assets.
For example, the metadata description for an image in the Lottie spec is as follows:

```json {6-7}
"assets": [
  {
    "id": "image_0",
    "w": 926,
    "h": 172,
    "u": "images/",
    "p": "img_0.png",
    "e": 0
  }
]
```

Here, the `u` and `p` fields respectively represent the relative path and file name of the image asset with `id` `image_0`.

Image assets can also come from a URL; in this case, set `u` to empty and put the full URL in `p`:

```json {6-7}
"assets": [
  {
    "id": "image_0",
    "w": 926,
    "h": 172,
    "u": "",
    "p": "https://example.com",
    "e": 0
  }
]
```

Additionally, Base64 inline images are supported. In this case, set `e` (Embedded) to `1` and put the Base64 data in `p`:

```json {6-7}
"assets": [
  {
    "id": "image_0",
    "w": 926,
    "h": 172,
    "u": "",
    "p": "data:image/png;base64,...",
    "e": 1
  }
]
```

In AnimaX, you can also inject asset resources using formatted strings via [`src-format`](../../../api/elements/built-in/animax-view.html#src-format) and [`src-polyfill`](../../../api/elements/built-in/animax-view.html#src-polyfill), or load resources on demand at runtime using [Dynamic Properties](../dynamic_property.mdx).

:::

::: details The src attribute in \<animax-view\>

The [`src`](../../../api/elements/built-in/animax-view.html#src) attribute accepts a URL that can point to a `.json` or `.zip` file.

- For `.json`: AnimaX will take the URL’s directory prefix as the asset root, and load assets by concatenating the relative paths from the spec with this root.
- For `.zip`: AnimaX uses the root directory within the archive as the asset root, parsing and loading assets according to the relative or absolute paths in the spec.

For simpler scenarios, you can pass complete animation data directly via the [`json`](../../../api/elements/built-in/animax-view.html#json) attribute; in this case (if assets are included), referencing external resources via relative paths is not supported.

:::
