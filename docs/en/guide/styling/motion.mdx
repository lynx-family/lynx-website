# @lynx-js/motion

`@lynx-js/motion` is the Lynx adapter for Motion. Use it when an interaction should update visual properties on the main thread without waiting for background-thread React updates.

It is a good fit for:

- Element animations that should start from React lifecycle code.
- Repeating animations that need `stop()` for cleanup or user input.
- Animating CSS-like values such as transforms, colors, units, and keyframes.
- Motion values that feed style effects or gesture calculations.

For the smallest possible main-thread animation runtime, use [Motion Mini](./motion-mini) instead.

## Install

```bash
pnpm add @lynx-js/motion
```

Use the regular package entry for the full adapter:

```ts
import { animate, motionValue, stagger } from '@lynx-js/motion';
```

Most `@lynx-js/motion` calls should run inside a function marked with `'main thread';`. From React lifecycle code on the background thread, schedule that function with `runOnMainThread()`.

## Choose the right API

Use CSS transitions or keyframes for static style changes declared in CSS. Use the built-in `lynx.animate()` API for small imperative effects that do not need Motion primitives. Use `@lynx-js/motion` when you need Motion-style controls, values, interpolation, spring options, or gesture-driven effects.

For numeric-only interactions where bundle size is the main concern, prefer [Motion Mini](./motion-mini).
