# Build-time Macros

ReactLynx's compiler plugin (`@lynx-js/react-rsbuild-plugin`, built on `@lynx-js/react-webpack-plugin`) replaces a set of **build-time macros** — special global identifiers that are substituted with constant values while your app is compiled. Because the values become constants, any code branch guarded by a macro that evaluates to `false` is removed by dead-code elimination, so it adds **no runtime cost and no bundle size**:

```tsx
if (__DEV__) {
  // This whole block is stripped from production builds.
  console.log('debug only');
}
```

:::tip
Every macro is also declared as a TypeScript global by `@lynx-js/react`, so you can use it directly without a `Cannot find name '__DEV__'` error.
:::

## Thread macros

ReactLynx runs your code across two threads — the **main thread** and the **background thread** (see [Main Thread Script](./main-thread-script.mdx)). These macros tell the compiler which thread a piece of code belongs to, so the bundle for the other thread can drop it entirely.

| Macro             | Type      | Meaning                                    |
| ----------------- | --------- | ------------------------------------------ |
| `__MAIN_THREAD__` | `boolean` | `true` in the main-thread bundle.          |
| `__BACKGROUND__`  | `boolean` | `true` in the background-thread bundle.    |
| `__LEPUS__`       | `boolean` | _Deprecated_ — alias of `__MAIN_THREAD__`. |
| `__JS__`          | `boolean` | _Deprecated_ — alias of `__BACKGROUND__`.  |

For example, a native bridge call only makes sense on the background thread, so guard it with `__BACKGROUND__` and it won't be emitted into the main-thread bundle:

```tsx
function App() {
  const showToast = __BACKGROUND__
    ? (message: string) => bridge.call('showToast', { message })
    : () => {};
  // ...
}
```

See the generated [built-in-macros](/api/react/Document.built-in-macros.mdx) reference for more examples.

## Mode macros

These macros toggle whole categories of development-time behavior. They are `true` in development and `false` in production, so dev-only instrumentation never ships.

### `__DEV__`

`true` in development, `false` in production (follows the build mode / `NODE_ENV`). Guards development-only code such as prop validation, invariant assertions and friendlier error messages. Everything inside `if (__DEV__) { … }` is removed from production builds.

### `__PROFILE__`

Enables performance instrumentation. When on, the runtime wraps key phases — hydration, rendering, and lifecycle events — with profiling marks (`profileStart` / `profileEnd`) that feed Lynx's trace and the [Profiler](./performance/profiling.mdx). It is on by default in development so you can profile locally, and off in production to avoid the overhead. Disable it in development with `REACT_PROFILE=false`, or force it with the plugin's `profile` option.

### `__REACT_DEVTOOL__`

Enables the dev-only runtime hooks that the Preact DevTools integration relies on (for example the snapshot ↔ element mapping). It is on by default in development. To use devtools in a **production** build, set `REACT_DEVTOOL=true` — which also keeps a user-imported `@lynx-js/preact-devtools` from being stripped out of the bundle.

## Logging macros

ALog is Lynx's native logging channel ("App Log"); messages sent through it appear in the platform's native logs even where `console.log` isn't surfaced. These macros are **off by default** because the output is verbose — turn them on only while debugging.

### `__ALOG__`

When enabled, the runtime emits detailed `[ReactLynxDebug]` diagnostics on **both threads** at key moments — for example the main-thread → background `OnLifecycleEvent` payload, and the snapshot trees before and after first-screen hydration. This is useful for debugging cross-thread data flow and hydration mismatches. Enable it with `REACT_ALOG=true`.

### `__ALOG_ELEMENT_API__`

Even more granular: it logs **every Element PAPI call** — the low-level element create / append / update operations the framework performs on the main thread — so you can see exactly how the element tree is built. This is very noisy; enable it with `REACT_ALOG_ELEMENT_API=true`.

## Plugin-option macros

These are derived from the ReactLynx plugin options rather than environment variables, and are mostly framework-internal:

| Macro                                                    | Type                         | What it does                                                                                                                                                                           |
| -------------------------------------------------------- | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `__ENABLE_SSR__`                                         | `boolean`                    | Gates server-side-rendering code (encode on the server, hydrate on the client). Set by `enableSSR`.                                                                                    |
| `__EXTRACT_STR__`                                        | `boolean`                    | Enables the "extract string" optimization that deduplicates string literals shared between the background (JS) and main-thread (Lepus) bundles to shrink output. Set by `extractStr`.  |
| `__FIRST_SCREEN_SYNC_TIMING__`                           | `'immediately' \| 'jsReady'` | Controls when the first screen is flushed — as early as possible (`immediately`) or after the background JS is ready (`jsReady`). Set by `firstScreenSyncTiming`.                      |
| `__GLOBAL_PROPS_MODE__`                                  | `'reactive' \| 'event'`      | Selects how `useGlobalProps` updates are delivered — reactive (context-based) or event-based. Set by `globalPropsMode`.                                                                |
| `__DISABLE_CREATE_SELECTOR_QUERY_INCOMPATIBLE_WARNING__` | `boolean`                    | Suppresses the development warning about deprecated `createSelectorQuery` / ref APIs that behave differently in ReactLynx 3.0. Set by `disableCreateSelectorQueryIncompatibleWarning`. |
| `__USE_ELEMENT_TEMPLATE__`                               | `boolean`                    | _Experimental_ — enables precompiled Element Templates. Set by `experimental_useElementTemplate`.                                                                                      |

## Environment variables

Several macros are resolved from environment variables read at build time:

| Variable                 | Macro                  | Example                                     |
| ------------------------ | ---------------------- | ------------------------------------------- |
| `REACT_PROFILE`          | `__PROFILE__`          | `REACT_PROFILE=false rspeedy build`         |
| `REACT_DEVTOOL`          | `__REACT_DEVTOOL__`    | `REACT_DEVTOOL=true rspeedy build`          |
| `REACT_ALOG`             | `__ALOG__`             | `REACT_ALOG=true rspeedy build`             |
| `REACT_ALOG_ELEMENT_API` | `__ALOG_ELEMENT_API__` | `REACT_ALOG_ELEMENT_API=true rspeedy build` |
