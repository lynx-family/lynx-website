# Agent Skills for Lynx

[Agent Skills](https://github.com/anthropics/skills) are reusable rule sets that enhance coding agents with domain-specific knowledge. They provide automated detection, best-practice guidance, and auto-fix capabilities directly inside your editor.

The Lynx community maintains a collection of skills at [lynx-community/skills](https://github.com/lynx-community/skills) to help coding agents understand Lynx's unique architecture and patterns.

## Available Skills

### reactlynx-best-practices

Teaches agents about Lynx's dual-thread architecture patterns. Includes static analysis rules and auto-fix for common issues:

- **`detect-background-only`** (Critical) — Ensures native API calls (`lynx.getJSModule`, `NativeModules`) only happen in background thread contexts, preventing UI blocking.
- **`proper-event-handlers`** — Correct event handler patterns (`bindtap`, `catchtap`) and propagation in ReactLynx.
- **`main-thread-scripts-guide`** — How to properly use `'main thread'` directive and `main-thread:` event prefix for smooth animations and gesture handling.
- **`hoist-static-jsx`** — Performance optimization by hoisting static JSX elements outside components to avoid unnecessary re-creation.

:::tip
These rules are also documented as a standalone guide. See [Best Practices](/react/best-practices) for detailed explanations and code examples.
:::

### devtool

A CLI-based skill that lets agents interact with Lynx DevTool to inspect and debug running Lynx applications on connected devices (Android, iOS, Desktop). The skill provides a Node.js CLI (`node scripts/index.mjs`) that agents invoke directly — no MCP server required.

:::info DevTool Skill vs DevTool MCP
There are two ways for coding agents to interact with Lynx DevTool:

- **DevTool Skill** (this page) — CLI-based, works with any coding agent (Claude Code, Cursor, Copilot), installed via `npx skills add`.
- **[DevTool MCP](/ai/lynx-devtool-mcp)** — MCP server protocol, works with MCP-compatible clients.

Both provide similar capabilities; choose the one that fits your agent setup.
:::

**DOM inspection** — Query elements with `querySelector`, retrieve the full document tree with `getDocument`, read `getOuterHTML` and `innerText` of any node.

**CSS inspection** — Read `getComputedStyle`, `getMatchedStyles`, and `getInlineStyles` for any element to understand applied styling.

**Console logs** — Read console output with level filters (log, warn, error) and range filters to focus on specific time windows.

**Screenshot capture** — Take screenshots of the current Lynx page for visual verification and multimodal analysis.

**Touch simulation** — Simulate touch events via `emulateTouchFromMouseEvent` for automated interaction testing (tap, drag, scroll).

**Page navigation** — Reload pages or redirect to a local dev-server URL during development workflows. Open arbitrary URLs in the connected Lynx app.

**JavaScript evaluation** — Evaluate JavaScript expressions directly in the Lynx runtime for inspecting state or triggering actions.

**Source listing** — List all parsed scripts loaded by the Lynx runtime, useful for mapping console stack traces back to source files.

**Troubleshooting** — Built-in debug logging for connector and transport issues. Enable debug namespaces to diagnose connection problems between the agent and DevTool.

### lynx-error-remap

Helps agents debug main-thread runtime errors by remapping minified error messages back to their original source, making stack traces actionable.

### lynx-ptrace

Enables agents to analyze Lynx performance traces, identifying bottlenecks in template loading, JS App loading, cross-thread calls, and rendering pipelines.

## Installation

Install the skills collection with a single command:

```bash
npx skills add lynx-community/skills
```

This adds the skill rules to your project so that compatible coding agents (such as Claude Code, Cursor, or Copilot) can automatically apply them.
