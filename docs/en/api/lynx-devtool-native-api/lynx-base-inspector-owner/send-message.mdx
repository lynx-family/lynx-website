# sendMessage

<APISummary />

Sends a debugging message of a specified type to the communication peer (e.g., the [DevTool Desktop Application](/guide/devtool#run-lynx-devtool-desktop-application)).

It works with the [`subscribeMessage`](/api/lynx-devtool-native-api/lynx-base-inspector-owner/subscribe-message) method to enable two-way communication.

## Syntax

### iOS

```objective-c title="LynxBaseInspectorOwnerNG.h"
- (void)sendMessage:(nonnull CustomizedMessage *)message;
```

#### Parameters

- `message`: The message to send, of type [`CustomizedMessage`](/api/lynx-devtool-native-api/customized-message).

### Android

```java title="LynxBaseInspectorOwnerNG.java"
void sendMessage(CustomizedMessage message);
```

#### Parameters

- `message`: The message to send, of type [`CustomizedMessage`](/api/lynx-devtool-native-api/customized-message).

### Harmony

```ts title="LynxBaseInspectorOwner.ets"
sendMessage(type: string, message: string): void;
```

#### Parameters

- `type`: The type of message to send, such as [CDP](https://chromedevtools.github.io/devtools-protocol/) or other custom message types.
- `message`: The message data to send.

## Usage Example

Please refer to [`subscribeMessage`](/api/lynx-devtool-native-api/lynx-base-inspector-owner/subscribe-message#usage-example).
