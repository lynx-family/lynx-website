# Document Contexts & Tags

## Document Context (Context)

Document context (`context`) is used to manage **sidebar filtering** for different types of documents. By setting context markers in the document metadata, sidebar items can be hidden or shown based on the context.

In the documentation sidebar metadata file (such as `_meta.json`), add the [`context` property](https://rspress.dev/guide/basic/auto-nav-sidebar#object) to the document item:

```json
{
  "context": "react"
}
```

Or in [frontmatter](https://rspress.dev/api/config/config-frontmatter#context):

```mdx
---
context: react
---
```

### Framework Type Markers

- `react` - Used to identify React-related documentation.
- `vue` - Used to identify Vue-related documentation (Vue is used here only as an example).

Documents marked with a framework type will be automatically shown or hidden in the sidebar by the DSL Switcher component.

At the same time, documents using framework markers need to follow the naming conventions below to achieve automatic redirection when switching with the DSL Switcher.

```mdx title="foo.react.mdx"
---
context: react
title: Foo
---

# Foo (ReactLynx)
```

```mdx title="foo.vue.mdx"
---
context: vue
title: Foo
---

# Foo (Vue)
```

Visit [Manipulating Elements](/guide/interaction/event-handling/manipulating-element.react.mdx) to see the actual effect.

## Sidebar Tags (Tag)

Sidebar tags (`tag`) are used to display badges in the sidebar, which is a [feature natively supported by Rspress](https://rspress.dev/guide/default-theme/tag).

In frontmatter, add the [`tag` property](https://rspress.dev/api/config/config-frontmatter#tag):

```mdx
---
tag: BTS
---
```

Or in `_meta.json`:

```json
{
  "tag": "BTS"
}
```

### Built-in Tags

Rspress has built-in tag types that automatically apply specific styles:

- `new` - Displays a green "New" badge.
- `experimental` - Displays a yellow "Experimental" badge (with icon).
- `deprecated` - Displays a red "Deprecated" badge (with icon).

### Custom Tags

Any non-built-in text will be displayed as a default style badge. Currently, we use the following custom tags:

- `BTS` - Identifies availability only in BTS (Background Thread Script).
- `MTS` - Identifies availability only in MTS (Main Thread Script).
- `XElement` - Identifies XElement related.
- `Android` - Identifies availability only on Android platform.
- `iOS` - Identifies availability only on iOS platform.
- `Web` - Identifies availability only on Web platform.
- `Harmony` - Identifies availability only on Harmony platform.
