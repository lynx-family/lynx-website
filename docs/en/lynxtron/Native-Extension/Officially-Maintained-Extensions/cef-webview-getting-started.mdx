# Using webview in Lynxtron

[`<webview>`](/api/elements/built-in/webview) is the Lynx element for embedding Web content. Its attributes, events, and methods are defined by the Lynx element API. This page only explains how to introduce this capability in Lynxtron through the officially maintained npm package.

Because CEF and WebView artifacts can significantly increase package size, Lynxtron does not bundle `<webview>` into the runtime by default. Instead, it is provided as an officially maintained Lynx native library. Other official Lynx components remain built in and can be used in Lynx UI normally.

The officially maintained `@lynx-js/cef-webview` package provides a native view capability based on the Chromium Embedded Framework (CEF). You can consume it like other Lynx native libraries: install the package, load its native library in the Lynxtron host process, initialize CEF in the main process, and then use `<webview>` in Lynx UI.

The native `<webview>` element is registered by the package C++ library through `LYNX_REGISTER_ELEMENT(...)`. Application code only needs to make sure the library is loaded and call `initialize()`; it should not call lower-level Lynx registration APIs.

## Install the npm Package

<PackageManagerTabs command="install @lynx-js/cef-webview" />

## Enable AutoLink

Enable `pluginLynxtronAutoLink()` in the Lynxtron host-side Rsbuild/Rspack config:

```ts
import { pluginLynxtronAutoLink } from '@lynx-js/lynxtron-autolink/rsbuild';

export default {
  target: 'electron-main',
  plugins: [pluginLynxtronAutoLink()],
};
```

AutoLink is the recommended helper for generating the host-side native library loader. The underlying model is still static registration plus loading the shared library in the Lynxtron host process.

## Initialize CEF in the Main Process

Import the package and initialize CEF before creating a `LynxWindow` or loading a Lynx bundle:

```typescript {1-2,5}
import { app, LynxWindow } from '@lynx-js/lynxtron';
import cefWebview from '@lynx-js/cef-webview';

app.whenReady().then(() => {
  cefWebview.initialize();

  const win = new LynxWindow({
    width: 1200,
    height: 800,
  });

  win.loadFile('main.lynx.bundle');
});
```

## Use It in Lynx UI

After the native library is loaded and CEF is initialized in the main process, you can use `<webview>` in Lynx UI.

```tsx
export function App() {
  return (
    <view style={{ width: '100%', height: '100%' }}>
      <webview
        src="https://www.example.com"
        style={{ width: '100%', height: '100%' }}
      />
    </view>
  );
}
```

For the attributes, events, and methods supported by `<webview>`, see the [`<webview>` API](/api/elements/built-in/webview).

## Packaging Notes

`@lynx-js/cef-webview` contains platform-specific native libraries and CEF binary artifacts. If you use AutoLink, keep it enabled for the packaged host build. If you use a custom loading or packaging flow, make sure the native library and CEF artifacts are included in the app package.
