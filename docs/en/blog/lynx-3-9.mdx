Lynx 3.9 is now officially released!

This release continues our monthly cadence with updates across **Lynx for AI**, [A2UI](/react/genui/a2ui), [Autolink](/guide/autolink), ReactLynx rendering and build optimization, Element capabilities, CSS compatibility, Lynx APIs, and DevTool debugging.

## Lynx for AI

Lynx 3.9 expands the runtime foundation for AI Agent generated interfaces.

We released Lynx A2UI, which connects model conversations, protocol messages, component catalogs, and ReactLynx rendering into a complete UI generation pipeline. Instead of asking models to generate arbitrary UI code, A2UI lets Agents compose interfaces within a controlled set of components and capabilities.

Read [Generative UI on Lynx with A2UI](/blog/lynx-a2ui) for the launch story, or jump straight to the [A2UI guide](/react/genui/a2ui) to start integrating it.

We are also improving how AI systems consume Lynx documentation, so Agents can better understand Lynx APIs, CSS, layout, Elements, and configuration instead of relying on assumptions from the Web platform.

## Autolink

Lynx 3.9 introduces [Autolink](/guide/autolink), an integration mechanism that lets host apps automatically discover native libraries installed under `node_modules` and register their Elements, Native Modules, and Services during Android and iOS initialization.

For app teams, this removes the need to wire every native library by hand. For library authors, providing a `lynx.lib.json` manifest and the corresponding native implementation is enough for Autolink to generate the registry that the host app needs at build time.

## ReactLynx

ReactLynx continues to reduce rendering overhead and improve bundle output in 3.9.

The multi-slot Snapshot optimization extends Preact support for multiple slots, reducing unnecessary Snapshot and wrapper nodes. It is enabled by default in `@lynx-js/react` 0.120.0.

On the build side, the ReactLynx toolchain now supports a multi-threaded Minimizer mechanism for more effective compression and tree-shaking across main-thread and background-thread bundles. You can enable the default optimization through [`optimizeBundleSize`](/api/rspeedy/react-rsbuild-plugin.pluginreactlynxoptions.optimizebundlesize), or configure thread-specific compression with [`Minify.mainThreadOptions`](/api/rspeedy/rspeedy.minify.mainthreadoptions) and [`Minify.backgroundOptions`](/api/rspeedy/rspeedy.minify.backgroundoptions).

ReactLynx also now supports [`createPortal`](/api/react/Function.createPortal), allowing content to render into a specified container node.

## Element Updates

This release extends several built-in and platform Elements.

### `<frame>`

[`<frame>`](/api/elements/built-in/frame) adds the `enable-multi-async-thread` boolean attribute to control whether embedded Lynx pages use multiple async threads.

It also adds `preset-height` and `preset-width` for specifying dimensions before content initialization, plus a `bindloadmetrics` event for tracking frame loading and performance metrics.

### `<image>`

[`<image>`](/api/elements/built-in/image) adds `region-to-decode` for region-based image decoding. This helps avoid unnecessary decode work when a page only needs to display part of a larger image.

### `<view>`

[`<view>`](/api/elements/built-in/view) adds `android-enable-auto-blur`, which controls whether CSS `filter: blur(...)` updates in real time on Android OS versions earlier than 12, helping optimize behavior on older Android versions.

### `<overlay>`

On HarmonyOS, [`<overlay>`](/api/elements/built-in/overlay) supports the `mode` attribute to control whether the current overlay layer is page-level (`page`) or top-level (`top`).

## CSS Updates

Lynx 3.9 adds several CSS capabilities:

- [`!important`](/api/css/selectors#important-support): available when `engineVersion >= 3.9`.
- `flexBasisZeroPercent`: when enabled, omitted `flex-basis` values in the `flex` shorthand are parsed as `0%`, matching behavior from some browsers.
- `enableGridPlacementShorthands`: enables [`grid-row`](/api/css/properties/grid-row) and [`grid-column`](/api/css/properties/grid-column) shorthand properties.
- Standard [`position: sticky`](/api/css/properties/position): enabled through `enableNewSticky`.
- HarmonyOS support for [`offset-path`](/api/css/properties/offset-path), [`offset-distance`](/api/css/properties/offset-distance), and [`offset-rotate`](/api/css/properties/offset-rotate).

## Gestures, APIs, and DevTool

On HarmonyOS, the new `ReuseNativeContent` interface lets clients forward event and gesture handling back to a Lynx page after reusing Lynx page nodes. The default value of the `enableNewIntersectionObserver` page flag is now `true`.

[`requestResourcePrefetch`](/api/lynx-api/lynx/lynx-request-resource-prefetch) gains a new `config` parameter with `awaitComplete` and `awaitTimeout`, so callbacks can run after prefetch fully completes.

For debugging, MTS debugging now supports RTSVM. On HarmonyOS, BTS debugging supports the case where a `LynxBackgroundRuntime` is attached to a `LynxView`.

## Performance

The 3.9 release-gate benchmark shows that on HarmonyOS, cases such as `image_animation` gain roughly 10% in `TotalTime` and `CPUTime`, mainly thanks to the new image strategy optimization.

## Upgrade Guide

To upgrade to Lynx 3.9, follow the official [integration guide](https://lynxjs.org/3.9/guide/start/integrate-with-existing-apps.html) and update your Lynx dependency versions.

Lynx 3.9 includes some breaking changes. Before upgrading, validate the affected platforms, `engineVersion`, page switches, and host integration path for your app.

Thanks to everyone trying new releases, filing issues, and helping the Lynx ecosystem move forward. We will keep the monthly releases smaller, focused, and easier to adopt.
