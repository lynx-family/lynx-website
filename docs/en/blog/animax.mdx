[AnimaX](https://github.com/lynx-family/animax) is now available as an
open-source animation engine for teams that need more than basic vector
playback. It is built in C++ around a self-rendering pipeline, is compatible
with Lottie, and adds first-class support for Alpha Video and mixed animation
layers.

AnimaX is also published in the
[lynx-family/animax](https://github.com/lynx-family/animax) repository, with
native Android and iOS examples available from the project. For Lynx apps, the
new [`<animax-view>`](/api/elements/built-in/animax-view) element brings the
same runtime into the component tree; this Lynx integration is coming soon.

## When Lottie Is Not Enough

Lottie made vector animation portable across mobile and web, but production
motion often reaches beyond pure vectors. Complex After Effects scenes may use
images, text, particles, glow, or video-like effects. Traditional Lottie
workflows can force teams to trade visual fidelity for file size, platform
consistency, or runtime performance.

AnimaX is designed for that gap. Existing Lottie JSON files can be parsed and
played directly, while richer assets can combine vector, image, text, and Alpha
Video layers in one animation.

## A Unified Renderer

AnimaX uses a unified C++ renderer powered by
[Skity](https://github.com/lynx-family/skity). Instead of delegating to
different platform renderers, the engine owns parsing, rendering, and playback
across iOS and Android. This keeps output consistent and lets expensive
graphics work move away from the main thread.

That architecture matters most when animations become part of product UI:
onboarding scenes, rewards, creator tools, commerce effects, game-like
interactions, and any place where animation needs to be both expressive and
repeatable.

## Mixed Animation Layers

Alpha Video is the key capability that unlocks a different class of effects.
With transparency preserved in the video layer, designers can bring in complex
visuals that are difficult to rebuild as pure vectors, then compose them with
normal Lottie layers.

The result is a practical motion pipeline:

- designers keep the After Effects workflow they already know;
- AFX exports Lottie-compatible artifacts, including Alpha Video layers when
  needed;
- AnimaX plays the exported assets through one engine;
- apps can still update animation properties at runtime through dynamic
  properties.

## Use It In Lynx Or Native Apps

In a Lynx page, AnimaX is exposed through `<animax-view>`. This Lynx integration
is coming soon:

```tsx
<animax-view
  className="animax-item"
  src="https://lf-lynx.tiktok-cdns.com/obj/lynx-artifacts-oss-sg/plugin/static/animax-starter.json"
  autoplay={true}
  loop={true}
/>
```

The `src` attribute can load a `.json` or `.zip` animation resource. For
simpler cases, animation data can also be provided directly through the `json`
attribute. See the [`<animax-view>` API](/api/elements/built-in/animax-view) for
properties, events, and runtime controls.

AnimaX can also be integrated directly as a native library. For platform-level
usage, see the [Android native API reference](/animax/reference/android) and
[iOS native API reference](/animax/reference/ios), which cover view creation,
resource loading, playback control, events, and status APIs.

## Try It Today
