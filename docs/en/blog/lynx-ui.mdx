## Native UX, Web Velocity

The component problem isn't solved. Not for native.

- Web offers shadcn/ui and Radix UI with great DX, composability, and customization. Still, the platform ceiling remains low.
- Native has high-fidelity UX (120fps scroll, physics, system gestures) but zero composability and zero iteration speed.
- This forces a compromise. Lynx UI is built to eliminate it.

> Lynx UI: ready-to-use components that preserve native user experience while delivering the velocity of the web.

## Built on Native Ground

Before any JS, the foundation matters.

- Lynx engine provides high-performance native primitives: `scroll-view`, `list`, `scroll-coordinator`
- These render to actual native views, enabling behaviors that are structurally impossible on mobile web
- For example, scroll-coordinator collapses the header before the inner content begins to scroll—a seamless behavior on mobile that isn't achievable on the web.
- This is the foundation of Lynx UI

## Main Thread Scripts (MTS): Interaction Without Latency

The hardest part of native-quality interaction is timing.

- Lynx uses a dual-thread model: the main thread handles graphics rendering, while the background thread runs JavaScript (JS).
- Normally, handling events in the background thread introduces a delay as messages pass between threads, which can make interactions feel slow and noticeable.
- MTS moves interaction logic to the main thread, resulting in zero-latency response to user input
- Swiper as example: `delta in, transform out`, per frame, synchronously on the main thread
- This enables fully programmable per-frame callbacks for properties such as rotation, scale, opacity, or any other adjustable property.

## Composability: Primitives, Not Black Boxes

Encapsulation can become a trap. We have learned from Radix.

- A fully encapsulated component transforms every product requirement into a prop, leading to a never-ending cycle.
- Expose the underlying primitives instead; let developers compose
- Dialog as case study: `DialogTrigger` · `DialogView` · `DialogBackdrop` · `DialogContent` · `DialogClose` · `DialogRoot`
- UI Variants: dynamic class names (`ui-open`, `ui-close`) assign CSS classes based on the component's state, allowing CSS to manage how elements transition between states
- One root class name drives the entire theme via CSS variables and scales to any design system

## Motion: Physics That Feel Real

The difference between good and great is in the curve.

- Motion curves encode physical intuition: a spring animation feels alive; a bezier curve feels designed
- Partnered with TikTok's design team to bring physics-based motion to more surfaces
- Adapted the Motion.js imperative API fully into Lynx
- **Same code. Same API. Web and native.**
- We're now sponsoring the Motion.js project

## LUNA: A Design Language for Lynx UI

We are obsessed with design. LUNA is the proof.

- Lynx UI components are headless: no opinions on visual style, fully open to your design system
- LUNA (Lynx UI Neo-Aesthetics) is a reference design system built on top of Lynx UI, used across all examples and demos
- Design token system: dark/light mode, color, spacing, motion timing, all codified
- Extensible: LUNA is the default; your design system is the override
