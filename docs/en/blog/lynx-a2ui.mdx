[A2UI](https://a2ui.org/) is an open protocol initiated by Google that enables AI Agents to safely generate rich interactive interfaces without executing arbitrary code. Since its release, it has gained significant traction in the Generative UI space, with renderers already available for Angular, Flutter, Lit, React, and more.

Today, we are officially releasing **Lynx A2UI** — an A2UI renderer built on Lynx. It brings truly native experiences to AI Agent-generated interfaces and can be embedded reliably into existing apps.

## Streaming Generation, Native Rendering

The A2UI protocol natively supports [streaming](https://a2ui.org/concepts/data-flow/):
the Agent sends UI fragments as it reasons, and the client renders them progressively.
This incremental interaction model eliminates blank waiting screens — users watch the interface take shape as the Agent produces output.

Lynx A2UI uses Lynx's [dual-thread architecture](/react/thinking-in-reactlynx) to power this capability:
A2UI protocol messages are parsed on the Background Thread, while UI updates are committed on the Main Thread — both running in parallel without blocking each other.
Even when the Agent outputs UI fragments at high frequency, the interface remains smooth and responsive.

## Customizable Catalog

The core mechanism of A2UI is the [Catalog](https://a2ui.org/concepts/catalogs/):
the client declares which components, functions, and parameters it supports, and the Agent can only choose from that authorized scope.
This keeps generation bounded while allowing rendered content to integrate seamlessly into existing apps and blend naturally with the native interface.

Lynx A2UI ships with a built-in [default component catalog](https://lynx-stack.dev/a2ui/#/a2ui/catalog) based on [lynx-ui](/lynx-ui/), covering common components such as buttons, lists, and dialogs — ready to use out of the box.
When the default catalog doesn't meet more complex requirements, developers can also [define their own components](https://lynx-stack.dev/guide/genui/a2ui/catalog-guide), registering business capabilities into the Catalog so the Agent can invoke them on demand during conversations.

## Flexible Interaction Handling

A2UI uses the [`Action` mechanism](https://a2ui.org/concepts/actions/) to handle user interactions:
components can describe user intent in a structured way instead of running arbitrary code.
Local actions such as navigation, analytics, or client Function calls can respond instantly, while actions that require further reasoning can still be delegated to the Agent.

Lynx A2UI exposes the `onAction` callback through the `<A2UI>` render container, giving developers fine-grained control over UI interactions:
you can forward user actions to the Agent, or consume them directly on the rendering side and drive updates locally.

## Try It Now

[Lynx A2UI Playground](https://lynx-stack.dev/a2ui) provides a complete interactive experience covering the full pipeline from model conversation to UI rendering.
No local setup required — open your browser, chat with the Agent, and watch the interface generate in real time. To experience native rendering on a real device, download [Lynx Explorer](/guide/start/quick-start.html#prepare-lynx-explorer) and scan the QR code on the page.

Read the [A2UI Overview](/react/genui/a2ui) to understand the overall design, or jump straight to the [Lynx A2UI Integration Guide](https://lynx-stack.dev/guide/genui/a2ui) to integrate it into your app.
If you encounter issues, have feature suggestions, or want to share your use cases, feel free to open an Issue or join the discussion on [GitHub](https://github.com/lynx-family/lynx-stack).

We believe that cross-platform frameworks in the AI era should do more than just run AI-generated code — they should provide Agents with a secure, efficient, and native channel for rendering interfaces. Lynx A2UI is our first step in that direction, and we look forward to exploring the future of Generative UI together with the community.
