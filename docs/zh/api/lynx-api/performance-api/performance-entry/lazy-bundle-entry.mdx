# LazyBundleEntry

通过[懒加载组件](/react/code-splitting.html#lazy-loading-components)，开发者可以按需进行渲染，从而提升用户体验。`LazyBundleEntry` 用来记录懒加载组件的性能数据，它继承自 [`PerformanceEntry`](/api/lynx-api/performance-api/performance-entry)。

## Example

该示例展示了如何获取 `LazyBundleEntry`。

## 实例属性

### [entryType](/api/lynx-api/performance-api/performance-entry#entryType)

```tsx
entryType: string;
```

性能事件的类型，当前类所有实例的值均固定为 `resource`。

### [name](/api/lynx-api/performance-api/performance-entry#name)

```tsx
name: string;
```

性能事件的名称，当前类所有实例的值均固定为 `lazyBundle`。

### componentUrl

```tsx
componentUrl: string;
```

用于加载懒加载组件的 URL 字符串。

### size

```tsx
size: number;
```

懒加载组件的资源大小，单位为 Byte。

### mode

```tsx
mode: string;
```

加载懒加载组件的模式，取值为： `preload` 或 `normal`。

### sync

```tsx
sync: boolean;
```

懒加载组件是否是同步渲染。

### loadSuccess

```tsx
loadSuccess: boolean;
```

加载懒加载组件是否成功。

### requireStart

```tsx
requireStart: number;
```

下载懒加载组件的开始时间戳。该时间戳是一个表示为浮点型的 Unix 时间戳（单位：毫秒）。

### requireEnd

```tsx
requireEnd: number;
```

下载懒加载组件的结束时间戳。该时间戳是一个表示为浮点型的 Unix 时间戳（单位：毫秒）。

### decodeStart

```tsx
decodeStart: number;
```

懒加载组件解码的开始时间戳。该时间戳是一个表示为浮点型的 Unix 时间戳（单位：毫秒）。

### decodeEnd

```tsx
decodeEnd: number;
```

懒加载组件解码的结束时间戳。该时间戳是一个表示为浮点型的 Unix 时间戳（单位：毫秒）。
