import { APIBadge, APITable, AndroidOnly, IOSOnly, VersionBadge } from '@lynx';
import frameData from '@lynx-js/lynx-compat-data/elements/frame.json';

## 属性

### `auto-height`

<APIBadge data={frameData.elements.frame} query="auto-height" />

```tsx
// @defaultValue: false
'auto-height'?: boolean;
```

使框架高度跟随嵌入的 Lynx 页面的内容高度。启用此功能后，嵌入的页面可以报告其内容大小，框架会将该值用作其测量高度。

### `auto-width`

<APIBadge data={frameData.elements.frame} query="auto-width" />

```tsx
// @defaultValue: false
'auto-width'?: boolean;
```

使框架宽度跟随嵌入的 Lynx 页面的内容宽度。启用此功能后，嵌入的页面可以报告其内容大小，框架会将该值用作其测量宽度。

### `data`

<APIBadge data={frameData.elements.frame} query="data" />

```tsx
data?: Record<string, unknown>;
```

将数据传递给框架内嵌套的 Lynx 页面。

### `enable-multi-async-thread`

<APIBadge data={frameData.elements.frame} query="enable-multi-async-thread" />

```tsx
// @defaultValue: false
'enable-multi-async-thread'?: boolean;
```

覆盖内嵌 Lynx 页面是否使用多个异步线程。未设置时，frame 会继承宿主页面的配置。

### `global-props`

<APIBadge data={frameData.elements.frame} query="global-props" />

```tsx
'global-props'?: Record<string, unknown>;
```

将
`globalProps`
 传递给框架中嵌入的 Lynx 页面。嵌入的页面可以通过
`lynx.__globalProps`
 读取它。

### `preset-height`

<APIBadge data={frameData.elements.frame} query="preset-height" />

```tsx
'preset-height'?: `${number}px` | `${number}rpx`;
```

设置内嵌 Lynx 页面收到初始化内容区域前使用的预设高度。

### `preset-width`

<APIBadge data={frameData.elements.frame} query="preset-width" />

```tsx
'preset-width'?: `${number}px` | `${number}rpx`;
```

设置内嵌 Lynx 页面收到初始化内容区域前使用的预设宽度。

### `src`

<APIBadge data={frameData.elements.frame} query="src" />

```tsx
src: string;
```

设置框架资源的加载路径。

## 事件

前端可以绑定相应的事件回调来监听元素的运行时行为，如下所示。

### `bindload`

<APIBadge data={frameData.elements.frame} query="bindload" />

```tsx
bindload = (e: FrameLoadEvent) => {};
```

| 字段 | 类型 | 是否可选 | 默认值 | 平台 | 版本 | 描述 |
| --- | --- | --- | --- | --- | --- | --- |
| statusCode | 数字 | 否 | – |  <AndroidOnly /> <IOSOnly /> <VersionBadge>3.6</VersionBadge> | 3.6 | 框架加载状态码。 |
| statusMessage | 字符串 | 否 | – |  <AndroidOnly /> <IOSOnly /> <VersionBadge>3.6</VersionBadge> | 3.6 | 框架加载状态消息。 |
| url | 字符串 | 否 | – |  <AndroidOnly /> <IOSOnly /> <VersionBadge>3.6</VersionBadge> | 3.6 | 框架加载的 URL。 |

绑定框架加载事件回调。

### `bindloadmetrics`

<APIBadge data={frameData.elements.frame} query="bindloadmetrics" />

```tsx
bindloadmetrics = (e: FrameLoadMetricsEvent) => {};
```

| 字段 | 类型 | 是否可选 | 默认值 | 平台 | 版本 | 描述 |
| --- | --- | --- | --- | --- | --- | --- |
| entry | FrameLoadMetricsEntry | 否 | – |  <AndroidOnly /> <IOSOnly /> <VersionBadge>3.9</VersionBadge> | 3.9 | 内嵌 Lynx 页面发出的原始 `loadBundle` 性能条目。 |
| mode | 字符串 | 否 | – |  <AndroidOnly /> <IOSOnly /> <VersionBadge>3.9</VersionBadge> | 3.9 | frame 的加载模式。 |
| url | 字符串 | 否 | – |  <AndroidOnly /> <IOSOnly /> <VersionBadge>3.9</VersionBadge> | 3.9 | frame 加载的 URL。 |

绑定 frame 加载指标事件回调。
