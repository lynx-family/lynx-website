# markState

<APISummary />

用关联的 dataProcessor 名称标记当前的 TemplateData。当此 TemplateData 在 [UpdateData](/api/lynx-native-api/lynx-view/update-meta-data) 中使用时，将根据此名称找到相应的 dataProcessor 进行数据预处理。

## 语法

### Android

```java
public void markState(String name);
```

#### 参数说明

- `name`: 标记的名称。

### iOS

```objc
- (void)markState:(NSString *)name;
```

#### 参数说明

- `name`: 标记的名称。

### Desktop (C++)

```cpp
void MarkState(const std::string& name);
```

#### 参数说明

- `name`: 与当前 `LynxTemplateData` 关联的数据处理器名称。

### Desktop (C API)

```c
void lynx_template_data_mark_state(lynx_template_data_t* data,
                                   const char* name);
```

#### 参数说明

- `data`: 要标记的 `lynx_template_data_t` 实例。
- `name`: 与当前模板数据关联的数据处理器名称。
