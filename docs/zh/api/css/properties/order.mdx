# order

## 介绍

`order` 属性规定了容器中的子元件在布局时的顺序。元件按照 `order` 属性的值的增序进行布局。拥有相同 `order` 属性值的元件按照它们在源代码中出现的顺序进行布局。

## 语法

```css
/* Numerical value including negative numbers */
order: 5;
order: -5;
```

### 取值

- `<number>`

[`<number>`](api/css/data-type/number.mdx) 表示子元件所在的次序组。**默认值**为 `0`。

## 与 Web 的区别

1. 不支持在 [`position: absolute/fixed/sticky`](./position.mdx) flex and grid container children 上使用。
2. Lynx 中 `order` 的使用不会影响元件的渲染顺序，而 Web 会。
