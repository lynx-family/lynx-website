# overflow

## 介绍

CSS 属性 overflow 控制当一个元件的子（后代）节点内容超过其大小后，是否接受其裁剪

:::caution

- 建议非必要 overflow 的节点显式设置 `overflow: hidden`，尤其是动画节点。Android 上 `overflow: visible` 可能会造成重绘区域变大

:::

## 语法

```css
overflow: visible;
overflow: hidden;
```

### 取值

- `visible` 子（后代）节点不接受剪裁，内容可以溢出

- `hidden` 子（后代）节点接受剪裁，内容无法溢出

- 默认值 `visible`

## 与 Web 的区别

不存在 `scroll` 和 `auto` 模式，对于 `scroll` 请使用 [`<scroll-view>`](api/elements/built-in/scroll-view)。

## 注意事项

- Android 系统默认会对 `overflow < 1` 的节点离屏绘制，这会导致此节点的 `overflow: visible` 无效，可以在该节点加上 `overlap={false}` 属性来关闭离屏绘制。
