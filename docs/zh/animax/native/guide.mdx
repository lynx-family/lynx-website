# 原生接入指引

这篇文档介绍如何在原生应用中直接嵌入 AnimaX。你将学习如何添加依赖、注入网络请求和图片加载等基础能力、创建 `AnimaXView`，以及通过代码控制动画播放。

## 添加依赖

AnimaX iOS SDK 通过 CocoaPods 发布。在 `Podfile` 中添加以下内容：

```ruby
platform :ios, '12.0'

target 'YourApp' do
  # 核心 + 渲染后端
  pod 'AnimaX', '<animax_version>', :subspecs => [
    'Core',
    'Framework',
    'RenderBackend/Skity',
    'Textra',      # 文本图层复杂排版、多语言需要
    'ThirdParty'
  ]

  # 渲染引擎
  pod 'skity', '<skity_version>'

  # 文本渲染
  pod 'LynxTextra', '<textra_version>'

  # 基础框架
  pod 'LynxBase/Framework', '<lynx_version>'
end
```

然后执行：

```bash
pod install
```

## 初始化 AnimaX

iOS 上的 AnimaX 不需要全局初始化步骤。框架在你使用 `AnimaXContext` 创建第一个 `AnimaXView` 时自动初始化。

服务注册可以在应用启动时完成，或者在创建第一个视图之前按需注册：

```objc
// 在 AppDelegate 中，或创建第一个 AnimaXView 之前
#import <AnimaX/AnimaXServiceCenter.h>

// 注册服务——见下一节
// [[AnimaXServiceCenter sharedInstance] registerService:@protocol(...) withImpl:...];
```

## 注入基础能力

AnimaX 使用服务注册模式来管理网络资源加载和图片解码等能力。如果不注册这些服务，AnimaX 只能从本地文件和应用包中加载资源。

### 网络资源加载器

实现 [`AnimaXLoaderProtocol`](./api/animax-loader-protocol) 来处理网络请求。以下是使用 `NSURLSession` 的示例：

```objc
// HttpLoader.h
#import <AnimaX/AnimaXLoaderProtocol.h>

@interface HttpLoader : NSObject <AnimaXLoaderProtocol>
@end

// HttpLoader.m
#import "HttpLoader.h"
#import <AnimaX/AnimaXLoaderResponse.h>

@implementation HttpLoader

- (void)handleRequest:(AnimaXLoaderRequest *)request
           completion:(AnimaXLoaderCompletionHandler)completion {
    NSURL *url = [NSURL URLWithString:request.url];
    NSURLSessionDataTask *task = [[NSURLSession sharedSession]
        dataTaskWithURL:url
        completionHandler:^(NSData *data, NSURLResponse *response, NSError *error) {
            if (error) {
                AnimaXLoaderResponse *resp =
                    [AnimaXLoaderResponse responseWithError:error];
                completion(resp);
                return;
            }
            AnimaXLoaderResponse *resp =
                [AnimaXLoaderResponse responseWithPayload:data];
            completion(resp);
        }];
    [task resume];
}

- (AnimaXLoaderScheme)getScheme {
    return AnimaXLoaderSchemeHttp;
}

@end
```

通过 [`AnimaXResourceFactoryService`](./api/animax-resource-factory-service) 注册加载器：

```objc
#import <AnimaX/AnimaXServiceCenter.h>
#import <AnimaX/AnimaXResourceFactoryService.h>

// 在 AppDelegate 中，或创建第一个视图之前
@interface MyResourceFactory : NSObject <AnimaXResourceFactoryService>
@end

@implementation MyResourceFactory
- (NSArray<id<AnimaXLoaderProtocol>> *)createAnimaXLoaders {
    return @[[[HttpLoader alloc] init]];
}
@end

[[AnimaXServiceCenter sharedInstance]
    registerService:@protocol(AnimaXResourceFactoryService)
           withImpl:[[MyResourceFactory alloc] init]];
```

### 图片加载服务

在 iOS 上，实现 [`AnimaXImageDecoderService`](./api/animax-image-service) 来提供自定义图片解码：

```objc
#import <AnimaX/AnimaXImageDecoderService.h>

@interface MyImageDecoderService : NSObject <AnimaXImageDecoderService>
@end

@implementation MyImageDecoderService

- (UIImage *)loadImageData:(NSData *)data {
    // 使用你的图片管线（SDWebImage、Kingfisher 等）进行解码。
    // 返回 nil 则回退到默认的 UIImage 解码。
    return [UIImage imageWithData:data];
}

@end
```

注册服务：

```objc
[[AnimaXServiceCenter sharedInstance]
    registerService:@protocol(AnimaXImageDecoderService)
           withImpl:[[MyImageDecoderService alloc] init]];
```

## 创建 AnimaXView 并播放

```objc
#import <AnimaX/AnimaXView.h>
#import <AnimaX/AnimaXContext.h>
#import <AnimaX/BaseAnimaXAbility.h>

// 1. 创建 ability（能力容器）
BaseAnimaXAbility *ability = [[BaseAnimaXAbility alloc] init];

// 2. 使用 ability 创建 context
AnimaXContext *context = [[AnimaXContext alloc] initWithAbility:ability];

// 3. 创建视图
AnimaXView *animaxView = [[AnimaXView alloc] initWithContext:context];
animaxView.frame = CGRectMake(0, 0, 300, 300);

// 4. 配置并加载
[animaxView setLoop:YES];
[animaxView setAutoplay:YES];
[animaxView setObjectfit:@"contain"];

// 从 URL 加载（需要网络加载器服务）
[animaxView setSrc:@"https://example.com/animation.json"];

// 或从应用包加载
NSString *path = [[NSBundle mainBundle] pathForResource:@"my_anim" ofType:@"json"];
[animaxView setSrc:[NSURL fileURLWithPath:path].absoluteString];

// 5. 添加到视图层级
[self.view addSubview:animaxView];
```

也可以使用便捷工厂方法处理默认情况：

```objc
AnimaXContext *context = [AnimaXContext contextWithDefaultAbility];
AnimaXView *animaxView = [[AnimaXView alloc] initWithContext:context];
```

## 基础播放控制

动画加载完成后（`onReady`），你可以控制播放：

```objc
// 从当前位置开始播放
[animaxView play];

// 在当前帧暂停
[animaxView pause];

// 从暂停位置恢复
[animaxView resume];

// 停止并重置到第一帧
[animaxView stop];

// 跳转到指定帧
[animaxView seekTo:30];

// 播放片段（第 15 帧到第 75 帧）
[animaxView playFrom:15 to:75];

// 检查是否正在播放
BOOL isPlaying = [animaxView isAnimating];

// 获取当前帧（可能为小数）
double currentFrame = [animaxView currentFrame];

// 获取总时长（毫秒）
double durationMs = [animaxView durationInMS];
```

### 配置播放属性

```objc
[animaxView setLoop:YES];              // 无限循环
[animaxView setLoopCount:3];           // 循环 3 次
[animaxView setSpeed:2.0];             // 2 倍速播放
[animaxView setAutoReverse:YES];       // 循环时来回播放
[animaxView setStartFrame:24];         // 起始帧
[animaxView setEndFrame:144];          // 结束帧
[animaxView setProgress:0.5];          // 初始位置（0.0 - 1.0）
[animaxView setMaxFrameRate:60];       // 限制帧率
[animaxView setObjectfit:@"contain"];  // 内容适配模式
```

### 监听事件

```objc
// 遵循 AnimaXAnimationListener 协议
@interface MyViewController () <AnimaXAnimationListener>
@end

@implementation MyViewController

- (void)viewDidLoad {
    [super viewDidLoad];
    // ... 创建 animaxView ...

    // 将自己添加为监听器（添加到 ability 上）
    BaseAnimaXAbility *ability = animaxView.getAnimaXContext.ability;
    [ability addAnimationListener:self];
}

- (void)onReady:(NSDictionary *)params {
    // 动画已加载；播放 API 就绪
    NSNumber *total = params[@"total"];
}

- (void)onStart:(NSDictionary *)params {
    // 播放已开始
}

- (void)onCompletion:(NSDictionary *)params {
    // 播放完成（非循环模式）
}

- (void)onRepeat:(NSDictionary *)params {
    // 循环结束，下一次已开始
}

- (void)onError:(NSDictionary *)params {
    NSString *msg = params[@"msg"];
    NSNumber *code = params[@"code"];
}

- (void)onUpdate:(NSDictionary *)params {
    // 帧更新——需要先订阅帧
    NSNumber *current = params[@"current"];
}

- (void)onFps:(NSDictionary *)params {
    NSNumber *fps = params[@"fps"];
}

- (void)onTapLayers:(NSDictionary *)params {
    NSArray *layers = params[@"layerList"];
}

@end
```

:::info

在 iOS 上，你可以直接在视图上添加事件监听器（`[animaxView addAnimationEventListener:self]`），也可以添加到 ability 上（`[ability addAnimationListener:self]`）。视图方法内部会转发到 ability，两种方式都有效。

:::

### 订阅帧更新

```objc
// 订阅特定帧
[animaxView subscribeUpdateEvent:0];
[animaxView subscribeUpdateEvent:30];
[animaxView subscribeUpdateEvent:60];

// 或订阅所有帧
NSMutableArray *frames = [NSMutableArray array];
for (int i = 0; i < totalFrames; i++) {
    [frames addObject:@(i)];
}
[animaxView subscribeUpdateEvents:frames subscribe:YES];

// 设置 FPS 报告间隔（毫秒）
[animaxView setFPSEventInterval:500];
```

## 内存管理

AnimaXView 是一个 `UIView`——当它从视图层级中移除并释放时会自动清理。如果需要显式清理，可以在 player 上调用 `release`：

```objc
- (void)dealloc {
    // 视图及其资源会自动清理
}
```

## 注入实例级能力

对于大多数应用，全局的 `AnimaXServiceCenter` 已经足够。如果你需要实例级能力注入，例如不同页面使用不同的图片加载策略——可以创建自定义 ability。

```objc
// CustomAbility.h
#import <AnimaX/BaseAnimaXAbility.h>

@interface CustomAbility : BaseAnimaXAbility
@end

// CustomAbility.m
#import "CustomAbility.h"
#import <AnimaX/AnimaXServiceCenter.h>

@implementation CustomAbility

- (instancetype)init {
    self = [super init];
    if (self) {
        // 注册仅对此视图实例有效的服务
        // [self registerService:@protocol(...) withImpl:...];
    }
    return self;
}

@end

// 使用自定义 ability
CustomAbility *ability = [[CustomAbility alloc] init];
AnimaXContext *context = [[AnimaXContext alloc] initWithAbility:ability];
AnimaXView *animaxView = [[AnimaXView alloc] initWithContext:context];
```

:::info

通过 `registerService` 在 ability 上注册的服务优先级高于 `AnimaXServiceCenter` 中的全局服务。

:::

## 添加依赖

AnimaX Android SDK 已发布至 Maven Central。在模块级 `build.gradle` 中添加以下依赖：

```groovy
dependencies {
    // 核心 SDK
    implementation "org.lynxsdk.lynx:animax-sdk:<animax_version>"

    // 文本渲染（可选，文本图层需要）
    implementation "org.lynxsdk.lynx:animax-textra:<animax_version>"

    // 原生运行时库
    runtimeOnly "org.lynxsdk.lynx:lynx-base:<lynx_version>"
    runtimeOnly "org.lynxsdk.lynx:skity-native:<skity_version>"
    runtimeOnly "org.lynxsdk.lynx:lynxtextra:<textra_version>"
}
```

如果你的应用需要支持多种 ABI 架构，请配置 `ndk` 块：

```groovy
android {
    defaultConfig {
        ndk {
            abiFilters "armeabi-v7a", "arm64-v8a", "x86_64"
        }
    }
    packagingOptions {
        pickFirsts = [
            "lib/arm64-v8a/libanimax.so",
            "lib/arm64-v8a/libanimax_textra.so",
            "lib/arm64-v8a/liblynxbase.so",
            "lib/arm64-v8a/libskity.so",
            "lib/arm64-v8a/liblynxtextra.so",
            "lib/arm64-v8a/libc++_shared.so",
        ]
    }
}
```

## 初始化 AnimaX

在 `Application.onCreate()` 中初始化 AnimaX，只需执行一次。此操作会加载原生库并设置应用上下文。

```java
public class MyApplication extends Application {
    @Override
    public void onCreate() {
        super.onCreate();

        // （可选）在初始化之前注册服务——见下一节
        // AnimaXServiceCenter.inst().registerService(...);

        // 初始化 AnimaX, 可以子线程中调用
        AnimaX.inst().init(this);
    }
}
```

:::tip

在调用 `AnimaX.inst().init()` **之前**注册服务（网络加载器、图片服务），确保第一个动画加载时这些服务已经可用。

:::

## 注入基础能力

AnimaX 使用服务注册模式来管理网络资源加载和图片解码等能力。如果不注册这些服务，AnimaX 只能从本地文件和应用包中加载资源。

### 网络资源加载器

实现 [`IAnimaXLoader`](./api/animax-loader-protocol) 来处理网络请求。以下是使用 OkHttp 的示例：

```java
public class HttpLoader implements IAnimaXLoader {
    private final OkHttpClient client = new OkHttpClient.Builder()
            .connectTimeout(30, TimeUnit.SECONDS)
            .readTimeout(30, TimeUnit.SECONDS)
            .build();

    @Override
    public void load(IAnimaXLoaderRequest request,
                     IAnimaXLoaderCompletionHandler completionHandler) {
        String uri = request.getUri();
        Request httpRequest = new Request.Builder().url(uri).build();

        client.newCall(httpRequest).enqueue(new Callback() {
            @Override
            public void onFailure(Call call, IOException e) {
                completionHandler.onComplete(
                    AnimaXLoaderResponse.createErrorResponse(e));
            }

            @Override
            public void onResponse(Call call, Response response)
                    throws IOException {
                if (!response.isSuccessful()) {
                    completionHandler.onComplete(
                        AnimaXLoaderResponse.createErrorResponse(
                            new Exception("HTTP " + response.code())));
                    return;
                }
                byte[] bytes = response.body().bytes();
                completionHandler.onComplete(
                    AnimaXLoaderResponse.createByteArrayResponse(bytes));
            }
        });
    }

    @Override
    public AnimaXLoaderScheme getScheme() {
        return AnimaXLoaderScheme.HTTP;
    }
}
```

在 `Application.onCreate()` 中通过 [`IAnimaXResourceFactoryService`](./api/animax-resource-factory-service) 注册加载器：

```java
AnimaXServiceCenter.inst().registerService(
    IAnimaXResourceFactoryService.class,
    new IAnimaXResourceFactoryService() {
        @Override
        public List<IAnimaXLoader> createAnimaXLoaders() {
            return Collections.singletonList(new HttpLoader());
        }
    }
);
```

### 图片加载服务

AnimaX 将图片加载委托给 [`IAnimaXImageService`](./api/animax-image-service)，让你可以复用应用中已有的图片管线（Fresco、Glide、Coil 等）。

```java
public class MyImageService implements IAnimaXImageService {
    @Override
    public boolean loadImage(IAnimaXLoaderRequest request,
                             IAnimaXLoaderCompletionHandler completionHandler) {
        // 在这里委托给你的图片库。
        // 如果该服务处理了请求，返回 true；
        // 返回 false 则回退到资源加载器链。
        String uri = request.getUri();
        // ... 加载 bitmap 并调用 completionHandler.onComplete(...)
        return false;
    }
}
```

在 `Application.onCreate()` 中注册：

```java
AnimaXServiceCenter.inst().registerService(
    IAnimaXImageService.class,
    new MyImageService()
);
```

## 创建 AnimaXView 并播放

### 代码创建

```java
AnimaXView animaxView = new AnimaXView(this);
animaxView.setAutoPlay(true);
animaxView.setLoop(true);
animaxView.setObjectFit(ObjectFit.CONTAIN);

// 从 URL 加载（需要网络加载器服务）
animaxView.setSrc("https://example.com/animation.json");

// 或从 assets 加载
animaxView.setSrc(UriUtil.fromLocalAsset("animations/my_anim.json"));

// 或从文件路径加载
animaxView.setSrc(UriUtil.fromLocalFile("/path/to/animation.json"));

// 添加到布局
FrameLayout container = findViewById(R.id.animation_container);
container.addView(animaxView,
    new FrameLayout.LayoutParams(
        ViewGroup.LayoutParams.MATCH_PARENT,
        (int) (280 * getResources().getDisplayMetrics().density)
    )
);
```

### 代码中使用

```xml
<com.lynx.animax.ui.AnimaXView
    android:id="@+id/animax_view"
    android:layout_width="match_parent"
    android:layout_height="280dp"
    app:animax_autoPlay="true"
    app:animax_loop="true"
    app:animax_objectFit="contain" />
```

然后在代码中：

```java
AnimaXView animaxView = findViewById(R.id.animax_view);
animaxView.setSrc("https://example.com/animation.json");
```

:::tip

`UriUtil.fromLocalAsset()` 和 `UriUtil.fromLocalFile()` 是辅助方法，用于为资源加载器构建格式正确的 URI 字符串。

:::

## 基础播放控制

动画加载完成后（`onReady`），你可以控制播放：

```java
// 从当前位置开始播放
animaxView.getPlayer().play();

animaxView.getPlayer().pause();

// 从暂停位置恢复
animaxView.getPlayer().resume();

// 停止并重置到第一帧
animaxView.getPlayer().stop();

// 跳转到指定帧
animaxView.getPlayer().seek(30);

// 播放片段（第 15 帧到第 75 帧）
animaxView.getPlayer().playSegment(15, 75);

// 检查是否正在播放
boolean isPlaying = animaxView.getPlayer().isAnimating();

// 获取当前帧（可能为小数）
double currentFrame = animaxView.getPlayer().getCurrentFrame();

// 获取总时长（毫秒）
double durationMs = animaxView.getPlayer().getDurationMs();
```

### 配置播放属性

```java
animaxView.setLoop(true);                       // 无限循环
animaxView.getPlayer().setLoopCount(3);         // 循环 3 次（0 = 无限）

animaxView.getPlayer().setStartFrame(24);       // 起始帧
animaxView.getPlayer().setEndFrame(144);        // 结束帧
animaxView.getPlayer().setProgress(0.5f);       // 初始位置（0.0 - 1.0）
animaxView.getPlayer().setKeepLastFrame(true);  // 停止后保持最后一帧
```

### 监听事件

```java
animaxView.getPlayer().addAnimationListener(new AnimationListenerAdapter() {
```

### 订阅帧更新


## 注入实例级能力

对于大多数应用，全局的 `AnimaXServiceCenter` 已经足够。如果你需要实例级能力，例如不同页面使用不同的图片加载策略——可以创建自定义 ability。

```java
public class MyCustomAbility extends NativeAbility {

    public MyCustomAbility() {
        super();
        // 注册仅对此视图实例有效的服务
        registerService(IAnimaXImageService.class, new MyScopedImageService());
    }
}

// 创建 context 时使用自定义 ability
AnimaXContext context = new AnimaXContext.Builder(
    new MyCustomAbility(), this
).build();
AnimaXView animaxView = new AnimaXView(context);
```

:::info

在 ability 上注册的服务优先级高于 `AnimaXServiceCenter` 中的全局服务，这允许实例级服务覆盖全局服务。

:::
