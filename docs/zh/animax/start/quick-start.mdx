在本文档中，我们将从一个 Lynx 项目出发，运行并验证一个 AnimaX 的最小可运行示例。

## 开始

<Steps>
### 准备 Lynx 项目

请准备一个 Lynx 项目与示例应用（Example App）。
如果你对此没有头绪，请参考[这个文档](/guide/start/quick-start)创建并运行你的第一个 Lynx 项目。

你可以通过 Lynx [`<animax-view>`](/api/elements/built-in/animax-view) 验证 AnimaX，也可以参考开源仓库提供的原生 [Android](https://github.com/lynx-family/animax/blob/develop/example/android/README.md) 与 [iOS](https://github.com/lynx-family/animax/blob/develop/example/darwin/ios/README.md) 示例指南体验 AnimaX。

:::tip

💡 本文档的快速开始部分基于 ReactLynx 展示接入流程，以便更快上手。

使用 AnimaX 并不要求先集成 Lynx。任意应用均可参考 [lynx-family/animax](https://github.com/lynx-family/animax) 仓库中的原生示例直接接入 AnimaX。

:::

### 准备动画

通常，设计师会在 AE 中借助 AFX 插件导出 Lottie 格式的资源。

若对该资源的产出方式感兴趣，请参阅[导出插件](../afx)，了解 AFX 的导出流程与规范。

::: details AnimaX如何组织Lottie动画素材？

对于任意动画资源，都会配套一个符合 [Lottie 协议](https://lottiefiles.github.io/lottie-docs/)
的 `.json` 文件，用于存储素材的元信息。
以图片为例，Lottie 协议对其元信息的描述如下：

```json {6-7}
"assets": [
  {
    "id": "image_0",
    "w": 926,
    "h": 172,
    "u": "images/",
    "p": "img_0.png",
    "e": 0
  }
]
```

其中，`u`和`p`字段分别表示`id`为`image_0` 的图片资源的相对路径与文件名。

图片素材也可以来自 URL，此时应将 `u` 置空，并在 `p` 中填入完整的 URL：

```json {6-7}
"assets": [
  {
    "id": "image_0",
    "w": 926,
    "h": 172,
    "u": "",
    "p": "https://example.com",
    "e": 0
  }
]
```

此外，还支持通过 Base64 内联图片，此时将 `e`（Embedded） 设为 `1`，并在 `p` 中放入 Base64 数据：

```json {6-7}
"assets": [
  {
    "id": "image_0",
    "w": 926,
    "h": 172,
    "u": "",
    "p": "data:image/png;base64,...",
    "e": 1
  }
]
```

在 AnimaX 中，你还可以通过 [`src-format`](/api/elements/built-in/animax-view.html#src-format)
与 [`src-polyfill`](/api/elements/built-in/animax-view.html#src-polyfill) 以格式化字符串注入素材资源，
或借助[动态属性](../dynamic-property)在运行时按需加载资源。

:::

### 开始开发

示例包已经包含共享样式 `src/App.css`。
在代码编辑器中打开 `src/basic/index.tsx`，重点关注高亮的 `<animax-view>` 代码块。

你已经成功运行了一个 AnimaX 的最小可运行示例！

::: details \<animax-view\>中的src属性

[`src`](/api/elements/built-in/animax-view.html#src) 属性接受一个 URL，可指向 `.json` 或 `.zip` 格式的文件。

- 对于 `.json`：AnimaX 会截取该 URL 的前缀目录作为资源根目录，并将协议中的相对路径与该根目录拼接后加载素材。
- 对于 `.zip`：AnimaX 以压缩包内的根目录作为资源根目录，按协议中的相对或绝对路径解析并加载资源。

若使用更简化的场景，可直接通过 [`json`](/api/elements/built-in/animax-view.html#json) 属性传入完整的动画数据；
此时（若包含素材）不支持通过相对路径引用外部资源。

:::
