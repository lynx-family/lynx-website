Lynx 3.7 已经正式发布！

本次版本正式支持 **MacOS** 和 **Windows** 桌面端平台，扩展了 **Lynx for AI** 的开源 Skills 与可观测性能力，并为 **ReactLynx** 带来了更简单的 External Bundle preset 和跨线程共享模块能力。

下面我们一起来看看 Lynx 3.7 带来了哪些新变化。

## 桌面端支持：MacOS 与 Windows

我们[此前计划](/blog/lynx-open-source-roadmap-2025.html)在 Lynx 3.5 中开源桌面端支持，[随后决定](/blog/lynx-3-5.html#one-more-thing-desktop-support-preview)进一步开源完整的桌面端技术栈。在 Lynx 3.7 中，这一愿景已成为现实。

Lynx 正式支持 **MacOS** 和 **Windows** 作为一级平台。你现在可以在原生桌面应用中构建和渲染 Lynx 视图，使用与移动端和 Web 端完全相同的前端技术栈和工具链。

### 完整的渲染引擎登陆桌面端

驱动 Lynx 移动端的同一套渲染引擎现已在 MacOS 和 Windows 上可用，具备近乎完整的特性对齐。

**CSS：97% 属性覆盖率。** Lynx 支持的 319 个 CSS 属性中，311 个已在桌面端可用，包括：

- **布局**：Flexbox、Grid、Linear Layout、完整盒模型、定位与相对定位。
- **样式**：排版、边框、阴影、渐变、蒙版、滤镜，以及 [CSS 自定义属性](/api/css/properties/css-variable)。
- **动画**：`animation`、`transition`、`@keyframes` 和完整的 transform 能力。

**元素：73% 属性覆盖率。** 246 个元素属性中已有 179 个在桌面端可用，MacOS 和 Windows 的支持范围保持一致。大多数内建元件已经可以直接在桌面端使用。

这意味着大多数为移动端构建的 Lynx 页面已经可以在桌面端渲染，前端代码通常只需极少修改甚至无需修改。

你可以在 [API Status Dashboard](/help/dashboard) 上查看每个 API 的详细兼容状态，现已包含 **MacOS** 和 **Windows** 列。

### 快速上手

桌面端的接入细节可直接查看[将 Lynx 集成到现有应用](/guide/start/integrate-with-existing-apps)。该页面现已包含与 Android、iOS、HarmonyOS、Web 并列的桌面端指南。

### 桌面端专属能力

在渲染能力对齐之外，Lynx 3.7 还增加了面向桌面环境的专属能力，包括鼠标事件和 CSS `cursor` 支持。

#### 鼠标事件

Lynx 3.7 在桌面端已支持大部分[鼠标事件](/api/lynx-api/event/mouse-event)能力。

```tsx
<view
  onMouseDown={(e) =>
    console.log(`按键 ${e.button} 位置 (${e.clientX}, ${e.clientY})`)
  }
  onMouseMove={(e) => handleHover(e)}
  onMouseLeave={() => resetState()}
/>
```

#### CSS `cursor`

Lynx 3.7 在桌面端补充了 [CSS `cursor`](/api/css/properties/cursor) 支持，方便你为交互式 UI 提供更明确的指针反馈。

```tsx
<view style={{ cursor: 'pointer' }}>
  <text>Hover me</text>
</view>
```

## SVG 可缩放矢量图形元件

Lynx 3.7 新增了独立的 `<svg>` 元件，为图标、图表和其他分辨率无关的图形场景提供跨平台的原生矢量渲染能力。

## 面向 AI 的 Lynx：开源 Skills

Lynx 3.7 延续了 3.6 中 **Lynx for AI** 的方向，把更多日常开发流程沉淀为可直接使用的开源 agent skills。

你现在可以在 [/ai](/ai) 相关资源之外，直接通过下面的命令安装社区维护的技能集合：

```bash
npx -y skills add lynx-community/skills -s <skill-name>
```

这批已发布的 skills 主要覆盖三类任务：

- **页面检查与调试**：DevTool Skill 与 [Lynx DevTool MCP](/ai/lynx-devtool-mcp)。
- **TypeScript 与 ReactLynx 指导**：环境配置、类型、事件处理与最佳实践。
- **追踪与模板工具链**：trace 录制、trace 分析、TASM codec 和字节码调试映射。

这一轮最大的提升在于可观测性：coding agent 不仅能看代码、写代码，也能直接检查运行中的页面、验证交互并缩短自调试链路。

## ReactLynx 更新

Lynx 3.7 为 ReactLynx 带来了两个更偏工程效率的更新：更简单的 External Bundle 配置，以及更顺手的跨线程逻辑复用方式。

### External Bundle Preset

ReactLynx 项目现在可以通过一个 preset 快速启用常见的 external bundle 配置：

```js
import { pluginExternalBundle } from '@lynx-js/external-bundle-rsbuild-plugin';

export default defineConfig({
  plugins: [
    pluginExternalBundle({
      externalsPresets: {
        reactlynx: true,
      },
      externals: {
        './App.js': 'comp-lib.template.js',
      },
    }),
  ],
});
```

如果需要进一步自定义配置，可以结合 [External Bundle 指南](/rspeedy/external-bundle) 和 [`pluginExternalBundle`](/api/rspeedy/external-bundle-rsbuild-plugin.pluginexternalbundle) API 文档使用。

### 跨线程共享模块

通过 [Import Attributes](/api/react/Document.import-attributes) 声明共享模块后，普通逻辑可以在两个 runtime 中直接复用：

```js
import { myHelper } from './utils.js' with { runtime: 'shared' };

function App() {
  useEffect(() => {
    myHelper();
  }, []);

  function handleScroll() {
    'main thread';
    myHelper();
  }
}
```

这类能力非常适合数学计算、格式化逻辑等纯逻辑代码。模块会同时打包到两个 runtime 中，但各线程内的状态仍然彼此隔离。
