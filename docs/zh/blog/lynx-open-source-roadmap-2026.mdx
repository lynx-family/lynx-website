去年，我们在TikTok的支持下将Lynx框架[开源](/blog/lynx-unlock-native-for-more)，我们收到了开发者和更广泛社区的宝贵反馈。我们还分享了[2025年路线图](/blog/lynx-open-source-roadmap-2025)，包括：保持一致的发布节奏、扩展多平台支持并开放更多核心功能。随着从 v3.2 到 v3.6 的持续发布，Lynx 已在以下方面取得进展：全面支持 Web 和 [OpenHarmony](/blog/lynx-harmony)；扩展 [UI 组件](https://github.com/lynx-family/lynx-ui)并提供更稳定的原生 API。借助这些 UI 组件和 API，社区中涌现出了多个新项目，如 [miso-lynx](https://github.com/haskell-miso/miso-lynx)、[vue-lynx](https://vue.lynxjs.org/) 等。 我们认识到构建一个优秀的开发者框架是一项长期的努力——尤其是在快速发展的AI时代。今天，我们很高兴分享2026年Lynx开源路线图。

### 1. 更快且更可预测的发布节奏

在过去一年中，Lynx 采用了双月发布周期。从 2026 年年中开始，我们将逐步过渡到每月发布一次。在开发和发布过程中，我们也将持续改进发布说明与升级指南，强化核心 API 的稳定性保障，并提升跨版本的长期可维护性。

| 稳定版本 | 3.8     | 3.9     | 4.0     | 4.1     | 4.2     | 4.3     | 4.4     | 4.5     |
| -------- | ------- | ------- | ------- | ------- | ------- | ------- | ------- | ------- |
| 发布时间 | 2026/05 | 2026/06 | 2026/07 | 2026/08 | 2026/09 | 2026/10 | 2026/11 | 2026/12 |

### 2. 拥抱 AI

AI 正在从根本上重塑软件的构建方式。我们相信，在开放环境中构建生态，对于框架跟上 AI 的发展节奏、并实现原生融合至关重要。因此在今年，我们将重点投入于更加稳定、结构化的 API 设计，以及高质量、对大模型友好的文档（[lynx-website](https://github.com/lynx-family/lynx-website)）、Agent Skills、开发工具与示例。同时，我们也将探索基于 Lynx 的生成式 UI 方案，因为我们认为这将成为 AI 时代跨平台框架的重要发展方向。我们的目标是让 Lynx 成为具备 AI 原生的跨平台框架之一。

### 3. 建设包括桌面系统的多平台能力

多平台能力始终是我们的核心使命，这不仅体现在渲染层，也体现在能力体系、API 设计以及开发者体验的一致性上。目前我们已经实现了对 Android、iOS、Web 和 [OpenHarmony](/blog/lynx-harmony) 的官方支持，同时也开源了面向 PC 平台（macOS / Windows）的渲染引擎 [Clay](https://github.com/lynx-family/lynx/tree/develop/clay)。在今年，我们将继续提升 PC（macOS / Windows）场景的生产可用性，并加强与行业生态的融合，例如支持 Electron 等兼容场景（Lynxtron）。

### 4. 提供更具产品化能力的解决方案

开源后开发者的大量反馈，希望能够提供产品化的解决方案以能够基于 Lynx 快速构建跨平台应用， 为应对这些需求，TikTok 已经开源了 [Sparkling](https://github.com/tiktok/Sparkling) 基础设施，我们将继续完善包括设备 API、路由等能力支持，提供应用构建脚手架。在框架层，我们将持续提供丰富的UI，包括[组件库](https://github.com/lynx-family/lynx-ui)、动画、MTS能力等。同时，我们会在支持的平台上进一步优化运行时效率和整体性能表现，并且将开放更多用于指标监控和性能分析的工具，结合 [Lynx DevTool](/guide/start/integrate-lynx-devtool) 建设为开发者提供一站式的跨平台应用开发、调试和部署体验。

### 5. 加强社区建设与协作

跨平台框架是一个由开源社区持续推动、长期演进的技术领域。Lynx 本身也深受这一生态的启发，自开源以来，我们已经看到基于 Lynx 框架产生了多项新的创新实践。我们将进一步加强与生态的协作，包括 [React](https://react.dev/)、[Vue.js](https://vuejs.org/)、[Motion](https://motion.dev/) 等项目，并持续完善 MiniApps 能力，使其成为真正可用于生产环境的解决方案。 此外，Lynx 工具链基于 [RStack](https://github.com/rstackjs) 构建。我们将持续与 RStack 社区紧密合作，为开发者提供高性能的工具链 [rspeedy](/rspeedy/) 和更优质的开发体验。
