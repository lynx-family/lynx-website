# Lynx 3.6:

<BlogAvatar list={['liujilong', 'huxpro', 'lynx']} />

Lynx 3.6 已经正式发布!

让我们一起来看看有哪些新特性！

## reactlynx-use

[reactlynx-use](https://github.com/lynx-community/reactlynx-use) 是一个专门为 ReactLynx 设计的 React 风格 hooks 库，不仅内置近 30 个经开源社区验证的实用 Hooks，如 useEffectOnce、useDebounce 等，还提供了 Lynx 双线程架构下的 Hooks 能力，主要包括：

- useVelocity：追踪主线程 Tap 事件的速度与方向。
- useTapLock：将主线程 Tap 事件限制在指定的方向与范围内。
- useMainThreadImperativeHandle：useImperativeHandle 的 Lynx 主线程版本。
- useEventListener：提供更灵活的事件监听能力，支持 exposure、disexposure 等。

## Lynx Docs MCP

Lynx Docs MCP 允许您的编码代理（例如 Gemini、Claude、Cursor 或 Copilot）访问 Lynx 文档， 以协助您完成开发任务。
因此，我们专门优化了 llms.txt， 它是文档站点的精简版本，专为大型模型阅读而优化。

## `<overlay>` 元件

`<overlay>` 是一个不参与页面布局的承载容器。它本身不占据任何空间，也不需要设置宽度和高度。

`<overlay>`被渲染时，其内部的内容完全脱离 Lynx 文档流，被提升到 Lynx 之外的渲染层级进行展示。

`<overlay>` 的唯一子节点默认以屏幕尺寸作为宽高基准，推荐将其视为一个独立的绘制容器，用于承载完整的蒙层内容。

## HarmonyOS 支持了 Standalone BackgroundRuntime

HarmonyOS 新增支持 Standalone BackgroundRuntime，部分对齐了 Android & iOS 现有的 Standalone BackgroundRuntime 能力。
在 HarmonyOS 可以借助 Standalone BackgroundRuntime 来实现独立运行的后台 JS 线程，执行一些不依赖渲染的任务，或者提前创建 BackgroundRuntime, 优化 LynxView 的初始化耗时。

## HarmonyOS 平台新增异步创建 TemplateBundle 的方法

在 Android 和 iOS 平台上，开发者可以使用 TemplateBundle.fromBundle 方法创建 TemplateBundle。该方法可以在任意线程执行。
在 HarmonyOS 平台上，ArkTS 逻辑只能在应用主线程运行。在 3.6 版本，LynxSDK 提供异步创建 TemplateBundle 的方法，以减少主线程 I/O 操作带来的阻塞时间。

- TemplateBundle.fromTemplateAsync
- TemplateBundle.fromTemplateAsyncWithOption

## iOS 平台 LynxView 新增 enableAutoLayout 能力

iOS 平台新增方法 [LynxView enableAutoLayout]。
开启后 LynxView 支持 iOS AutoLayout 机制，从客户端约束中解析中 Lynx 排版所需的排版参数，不需要再手动设置 LynxView 排版参数。
