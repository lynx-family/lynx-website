## 原生基础，Web 速度

[内置元素](/guide/ui/elements-components.html#built-in-elements)和 [XElement](/guide/ui/elements-components.html#xelement) 为 Lynx 提供了至关重要的底层能力：流畅滚动、精准手势、[原生渲染](/guide/ui/elements-components.html#behind-the-elements-native-rendering)。但产品不是仅靠原子就能构建的，这正是[组件](/guide/ui/elements-components.html#components-composition-of-elements)的意义所在。在 Lynx 中，原生提供基础；源自 Web 的前端技术栈提供组合、状态、样式和产品特定行为的扩展层。

lynx-ui 落地了这一分层思路：**保留原生用户体验，提供 Web 级别的开发速度。**

## 可组合的原语

过度封装是一个陷阱。lynx-ui 不把组件当作 prop 不断膨胀的黑盒。相反，它提供结构化的原语，让团队按自己的产品需求自由组合。

[`Popover`](/lynx-ui/components/popover) 展示了这个思路。它不是一个不透明的单体组件，而是职责的组合：`PopoverRoot` 管理状态，`PopoverTrigger` 定义意图，`PopoverPositioner` 处理定位，`PopoverContent` 保持结构独立。

## LUNA

同样的可组合模型也为设计系统奠定了基础。lynx-ui 组件默认是[无头的](/lynx-ui/styling-theming)，不预设视觉风格。[LUNA](/lynx-ui/luna-themes-tokens) 是在此基础上构建的参考设计语言。

LUNA 从 Lynx 标志性的渐变色出发，将品牌表达调适为更柔和的 UI 色调：温暖的粉红和玫瑰色渐变至薰衣草和水蓝色。在 LUNA 中，渐变作为一等视觉材质，用于表达形态和运动，赋予组件一种流动介质的质感。这种细腻也延伸到了 token 命名：LUNA 描述的是表面的感受，而非其位置——`ambient` 融入背景，`faint` 处于感知的阈值，`veil` 和 `film` 创造柔和的半透明层。

## 可编程的交互

手势驱动的组件同时需要两件事：高度可定制的行为和低延迟的响应。滑动、拖拽、轻扫都是逐帧评判的。如果视觉反馈跟不上手势，组件可能仍然可用，但它不再有原生的感觉。

Lynx 的[双线程模型](/blog/lynx-unlock-native-for-more#use-the-main-thread-responsibly-for-interactivity)默认将 JavaScript 运行在背景线程，保持主线程专注于渲染。而[主线程脚本](/react/main-thread-script.html)可以在需要时将交互逻辑移至主线程，让逐帧行为完全可编程。[`Swiper`](/lynx-ui/components/swiper) 直接体现了这个思路：拖拽时卡片逐帧跟随手势，而不是事后追赶。

## Motion

[Motion](https://motion.dev/) 是 Web 社区最知名的动画库之一，以弹簧和贝塞尔曲线过渡著称。得益于[主线程脚本（Main-Thread Script）](/react/main-thread-script.html)带来的可编程性，我们将 Motion 适配到了 Lynx，即 `@lynx-js/motion`，一个直接复用 Motion 源码的薄适配层，其实验版本随 lynx-ui 一起提供。如果你在 Web 上用过 Motion，在 Lynx 中也同样适用。

[`Sheet`](/lynx-ui/components/sheet) 展示了这种配合的价值。拖拽、阻尼、释放和归位需要呈现为一个连续的行为。[主线程脚本](/react/main-thread-script.html)让交互紧贴渲染，Motion 定义面板的运动方式，二者结合让面板给人一种被"握住"的感觉，而非简单地回放动画。

## 立即体验

lynx-ui 目前已支持 iOS、Android 和 HarmonyOS，Web 和 Desktop 支持也在推进中。阅读[简介](/lynx-ui/)了解全貌，或直接跳到[快速开始](/lynx-ui/introduction.html#quick-start)安装 [`@lynx-js/lynx-ui-*`](https://www.npmjs.com/package/@lynx-js/lynx-ui)。

想要看看组件的实际效果，下载 [Lynx Explorer](/guide/start/quick-start.html#prepare-lynx-explorer)，扫描[组件示例](/lynx-ui/components/button)中的二维码，即可看到它们运行起来。也可以在你自己的 Lynx 应用中集成，在真实产品流程中试用。

我们希望 lynx-ui 能在真实应用的实践中不断成长。如果它体验很好、阻碍了你、或者缺少了什么，请在 [lynx-family/lynx-ui 仓库](https://github.com/lynx-family/lynx-ui)中告诉我们。一起塑造它的未来。
