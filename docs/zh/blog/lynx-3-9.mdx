Lynx 3.9 已经正式发布！

这个版本继续沿着月度发布节奏推进，重点包括 **Lynx for AI**、[Autolink](/guide/autolink)、ReactLynx 构建与渲染优化、元件能力扩展、CSS 标准能力补齐等。

下面我们一起来看看 Lynx 3.9 带来了哪些新变化。

## Lynx for AI

新增 lynx-api-docs agent Skill，优化 AI 使用 Lynx 文档的方式，让 Agent 更容易获得 Lynx API、CSS、布局、元件和配置相关知识，减少“按 Web 习惯写 Lynx 代码”带来的误用。

## Autolink

Lynx 3.9 引入了 [Autolink](/guide/autolink) 机制，让宿主应用可以自动发现安装在 `node_modules` 中的原生库，并在 Android 和 iOS 初始化时统一注册这些库提供的[元件](/guide/ui/elements-components)、[原生模块](/guide/use-native-modules)和 [Service](/api/lynx-native-api/lynx-service)。

对应用侧来说，这意味着接入一个原生库时不再需要逐库手动串联注册逻辑；对库作者来说，只要按照约定提供 `lynx.lib.json` 和对应原生实现，Autolink 就能在构建期生成宿主应用需要的 registry。

## ReactLynx

ReactLynx 在 3.9 中继续优化渲染层级和构建产物。

多 Slot Snapshot 优化通过扩展 Preact 的实现，减少不必要的 Snapshot 和 wrapper 节点数量，并已在 `@lynx-js/react` 0.120.0 中默认启用。

构建侧新增了双线程 Minimizer 机制，用于对主线程和后台线程产物进行更细粒度的压缩与裁剪。开发者可以通过 [`optimizeBundleSize`](/api/rspeedy/react-rsbuild-plugin.pluginreactlynxoptions.optimizebundlesize) 快速启用默认优化，也可以分别配置 [`Minify.mainThreadOptions`](/api/rspeedy/rspeedy.minify.mainthreadoptions) 和 [`Minify.backgroundOptions`](/api/rspeedy/rspeedy.minify.backgroundoptions)。

此外，ReactLynx 现在支持 [`createPortal`](/api/react/Function.createPortal)，让内容可以渲染到指定容器节点中。

## 元件能力

这个版本也扩展了多个内置元件和平台元件能力。

### `<frame>`

[`<frame>`](/api/elements/built-in/frame) 新增 `enable-multi-async-thread`，可控制嵌入式 Lynx 页面是否使用多个异步线程。

它也新增了 `preset-height` 和 `preset-width`，用于在内容初始化前指定尺寸；同时新增 `bindloadmetrics` 事件，方便宿主或业务侧跟踪 frame 的加载与性能指标。

### `<image>`

[`<image>`](/api/elements/built-in/image) 新增 `region-to-decode`，支持按图片区域解码。对于只需要展示局部区域的大图场景，这可以帮助减少不必要的解码成本。

### `<overlay>`

HarmonyOS 端的 [`<overlay>`](/api/elements/built-in/overlay) 支持 `mode` 属性，控制当前 overlay 的层级是页面级（`page`）还是顶层（`top`）。

## CSS 能力

Lynx 3.9 补齐了多项 CSS 能力。

- [`!important`](/api/css/selectors#important-support) 支持：配置 `engineVersion >= 3.9` 后可启用。
- `flexBasisZeroPercent` 开关：开启后，`flex` 简写中省略的 `flex-basis` 会按 `0%` 解析，用于对齐部分浏览器行为。
- `enableGridPlacementShorthands` 开关：开启后可以使用 [`grid-row`](/api/css/properties/grid-row) 和 [`grid-column`](/api/css/properties/grid-column) 简写。
- 标准 [`position: sticky`](/api/css/properties/position) 支持：可通过 `enableNewSticky` 开关启用。
- HarmonyOS 支持 [`offset-path`](/api/css/properties/offset-path)、[`offset-distance`](/api/css/properties/offset-distance) 和 [`offset-rotate`](/api/css/properties/offset-rotate)。

## Lynx API 与 DevTool

[`requestResourcePrefetch`](/api/lynx-api/lynx/lynx-request-resource-prefetch) 新增 `config` 参数，支持 `awaitComplete` 和 `awaitTimeout`。如果你希望 prefetch 完全结束后再触发 callback，现在可以直接通过配置表达。

DevTool 方面，HarmonyOS 上的 BTS 调试也支持 `LynxBackgroundRuntime` attach 到 `LynxView` 的场景。

## 性能表现

HarmonyOS 端通过新的图片策略优化，在 `image_animation` 等场景中将 `TotalTime` 和 `CPUTime` 降低约 10%。

## 升级指南

参照官网[接入 Lynx 到现有应用](https://lynxjs.org/3.9/zh/guide/start/integrate-with-existing-apps.html)，更新 Lynx 依赖版本，即可完成 Lynx 3.9 版本升级。

Lynx 3.9 包含部分破坏性变更。升级前请结合你的目标平台、`engineVersion`、页面开关和宿主集成方式进行验证。
