# 编译时宏

ReactLynx 的编译插件（`@lynx-js/react-rsbuild-plugin`，基于 `@lynx-js/react-webpack-plugin`）会在编译你的应用时替换一组**编译时宏**——一些特殊的全局标识符，会在编译期间被替换为常量值。由于这些值是常量，任何被取值为 `false` 的宏所守卫的代码分支都会被 dead-code elimination 移除，因此它们**不产生任何运行时开销，也不增加包体积**：

```tsx
if (__DEV__) {
  // 整段代码会在生产构建中被移除。
  console.log('debug only');
}
```

:::tip
每个宏都由 `@lynx-js/react` 声明为 TypeScript 全局变量，因此你可以直接使用它们，而不会出现 `Cannot find name '__DEV__'` 的报错。
:::

## 线程宏

ReactLynx 会把你的代码分布到两个线程上运行——**主线程**与**后台线程**（参见 [主线程脚本](./main-thread-script.mdx)）。这些宏告诉编译器某段代码属于哪个线程，从而让另一个线程的产物可以将其整段移除。

| 宏                | 类型      | 含义                                 |
| ----------------- | --------- | ------------------------------------ |
| `__MAIN_THREAD__` | `boolean` | 在主线程产物中为 `true`。            |
| `__BACKGROUND__`  | `boolean` | 在后台线程产物中为 `true`。          |
| `__LEPUS__`       | `boolean` | _已废弃_——`__MAIN_THREAD__` 的别名。 |
| `__JS__`          | `boolean` | _已废弃_——`__BACKGROUND__` 的别名。  |

例如，原生 bridge 调用只在后台线程上有意义，用 `__BACKGROUND__` 守卫它，它就不会被打进主线程产物：

```tsx
function App() {
  const showToast = __BACKGROUND__
    ? (message: string) => bridge.call('showToast', { message })
    : () => {};
  // ...
}
```

更多示例参见自动生成的 [built-in-macros](/api/react/Document.built-in-macros.mdx) 参考文档。

## 模式宏

这些宏用于整体开启或关闭某一类开发期行为。它们在开发环境为 `true`、生产环境为 `false`，因此仅用于开发的插桩代码永远不会进入产物。

### `__DEV__`

开发环境为 `true`，生产环境为 `false`（跟随构建模式 / `NODE_ENV`）。用于守卫仅开发期的代码，例如 prop 校验、不变量断言、更友好的错误信息。`if (__DEV__) { … }` 内的所有内容都会在生产构建中被移除。

### `__PROFILE__`

开启性能插桩。开启时，运行时会用性能标记（`profileStart` / `profileEnd`）包裹关键阶段——hydration、渲染、生命周期事件——这些标记会接入 Lynx 的 trace 与 [Profiler](./performance/profiling.mdx)。开发环境默认开启，方便你本地做性能分析；生产环境默认关闭，以避免额外开销。可在开发环境用 `REACT_PROFILE=false` 关闭，或用插件的 `profile` 选项强制开启。

### `__REACT_DEVTOOL__`

开启 Preact DevTools 集成所依赖的、仅开发期的运行时钩子（例如 snapshot ↔ element 的映射）。开发环境默认开启。若要在**生产**构建中使用 devtools，设置 `REACT_DEVTOOL=true`——它同时会保留用户引入的 `@lynx-js/preact-devtools`，使其不被从产物中裁剪掉。

## 日志宏

ALog 是 Lynx 的原生日志通道（"App Log"）；通过它输出的消息会出现在平台的原生日志里，即使在 `console.log` 不可见的环境中也是如此。这些宏**默认关闭**，因为输出非常冗长——仅在调试时开启。

### `__ALOG__`

开启后，运行时会在关键时刻于**两个线程**上输出详细的 `[ReactLynxDebug]` 诊断信息——例如主线程 → 后台线程的 `OnLifecycleEvent` 数据，以及首屏 hydration 前后的 snapshot 树。这对调试跨线程数据流与 hydration 不一致很有用。用 `REACT_ALOG=true` 开启。

### `__ALOG_ELEMENT_API__`

更细粒度：记录**每一次 Element PAPI 调用**——即框架在主线程上执行的底层元素 create / append / update 操作——让你能看清元素树究竟是如何被构建的。输出极为冗长；用 `REACT_ALOG_ELEMENT_API=true` 开启。

## 插件选项宏

这些宏来源于 ReactLynx 插件选项而非环境变量，且大多由框架内部使用：

| 宏                                                       | 类型                         | 作用                                                                                                                                                   |
| -------------------------------------------------------- | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `__ENABLE_SSR__`                                         | `boolean`                    | 守卫服务端渲染相关代码（服务端 encode、客户端 hydrate）。由 `enableSSR` 决定。                                                                         |
| `__EXTRACT_STR__`                                        | `boolean`                    | 开启「字符串提取」优化，对后台（JS）产物与主线程（Lepus）产物之间共享的字符串字面量去重，以减小产物体积。由 `extractStr` 决定。                        |
| `__FIRST_SCREEN_SYNC_TIMING__`                           | `'immediately' \| 'jsReady'` | 控制首屏何时刷新——尽可能早（`immediately`）或等后台 JS 就绪后（`jsReady`）。由 `firstScreenSyncTiming` 决定。                                          |
| `__GLOBAL_PROPS_MODE__`                                  | `'reactive' \| 'event'`      | 选择 `useGlobalProps` 更新的派发方式——响应式（基于 context）或基于事件。由 `globalPropsMode` 决定。                                                    |
| `__DISABLE_CREATE_SELECTOR_QUERY_INCOMPATIBLE_WARNING__` | `boolean`                    | 抑制关于已废弃的 `createSelectorQuery` / ref API（在 ReactLynx 3.0 中行为不同）的开发期警告。由 `disableCreateSelectorQueryIncompatibleWarning` 决定。 |
| `__USE_ELEMENT_TEMPLATE__`                               | `boolean`                    | _实验性_——开启预编译的 Element Template。由 `experimental_useElementTemplate` 决定。                                                                   |

## 环境变量

部分宏由构建时读取的环境变量决定：

| 变量                     | 宏                     | 示例                                        |
| ------------------------ | ---------------------- | ------------------------------------------- |
| `REACT_PROFILE`          | `__PROFILE__`          | `REACT_PROFILE=false rspeedy build`         |
| `REACT_DEVTOOL`          | `__REACT_DEVTOOL__`    | `REACT_DEVTOOL=true rspeedy build`          |
| `REACT_ALOG`             | `__ALOG__`             | `REACT_ALOG=true rspeedy build`             |
| `REACT_ALOG_ELEMENT_API` | `__ALOG_ELEMENT_API__` | `REACT_ALOG_ELEMENT_API=true rspeedy build` |
